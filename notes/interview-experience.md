**核心工作**：

## 2025 年 3 月

### 深信服（长沙）· 算法岗（安全方向）
**面试时间**：2025 年 3 月 18 日

**一面问题**：
- **QLoRA、AdaLoRA 原理？**
  - QLoRA：在 4-bit 量化模型上做 LoRA 微调，通过 NF4 量化 + 双重量化大幅降低显存；AdaLoRA：自适应分配 LoRA 秩——重要层给更多秩，次要层给更少秩，在总参数量不变下提升效果
- **项目三实习：Long-CoT 的 Prompt 如何设计？**
  - 使用结构化 prompt 引导模型先在 think 标签内分析客户意图、情绪和产品匹配度，再在 response 标签内给出销售话术
- **MoE 路由方式？**
  - 常见的 top-k 路由（每个 token 选择得分最高的 k 个专家）、Hash 路由（按 token 哈希值分配）、Expert Choice 路由（让专家主动选择 token）

**二面问题**：
- **业务面，围绕项目细节展开**
- **是否有验证 Long-CoT 思考过程的质量？如何评估？**

### 卓越教育（广州）· 算法岗
**面试时间**：2025 年 3 月 19 日

**一面问题**：
- **交叉熵的实现？**
  - 对模型输出的 logits 做 Softmax 得到概率分布，取目标类别的负对数：L = -log(p_target)。实现时注意数值稳定性（log_softmax 比 softmax + log 更稳定）
- **Focal Loss 的实现？**
  - FL = -(1-pₜ)ᵞ·log(pₜ)，其中 γ 控制易样本的降权程度。在交叉熵基础上乘以 (1-pₜ)ᵞ，使模型更关注难分类样本，缓解类别不平衡

### 科大讯飞（广州南沙）· 大模型算法岗（教育 / 医疗方向）
**面试时间**：2025 年 3 月 26 日

**一面问题**：
- **DeepSeek R1 的奖励函数设计，是否读过相关论文？**
  - R1 使用极简的规则 Reward：accuracy reward（答案正确+1）+ format reward（格式正确+0.5）。没有使用复杂的 LLM-as-Judge，证明了简单 Reward 配合 GRPO 也能涌现强推理能力
- **除了 length、format，还有哪些奖励维度？**
  - 内容质量（准确性、完整性、流畅度）、任务完成度、安全性（不产生有害内容）、多样性（避免重复）、风格匹配度
- **DPO、PPO、GRPO 在实际业务场景落地的优缺点？**
  - DPO：实现简单、无需在线采样，但依赖偏好数据质量且无法探索新策略；PPO：灵活但复杂（需四个模型）、训练不稳定；GRPO：简单稳定（无 Critic）、显存省，但对采样量敏感
- **业务场景题：如何消除大模型的幻觉？如何验证某段文本是否为幻觉？**
  - **自己思路**：多个模型投票判决（ensemble 降低误判）
  - **面试官思路**：在模型层面解决——使用 probing 探针，在首 token 后接一个线性层做二分类，判断生成的 token 是否为幻觉。本质是利用模型内部表征做自我检测
- **判断文本语言一致性问题的代码实现**：对文本按固定长度切片，分别统计每个切片中英文字符占比，任意切片超过阈值则判定为语言不一致，删除整段返回 false

**业务介绍**：
- 医疗、教育方向：作文批改、课件生成、辅助学习、全科目覆盖、司法认知、智慧城市
- 学习机 ToC 业务
- 技术组负责基座模型，教育方向大模型，深耕垂直领域

## 2025 年 4 月

### 字节跳动 · 算法岗（一面）
**面试时间**：2025 年 4 月 8 日

**面试问题**：
- **SFT 损失和强化学习损失的区别？**
  - SFT 使用交叉熵损失，逐 token 计算与教师标签的差异，目标是模仿；RL（如 GRPO/PPO）使用策略梯度类损失，通过 Reward 信号引导模型探索更好的生成策略。SFT 是"学别人怎么做"，RL 是"自己试，做好有奖励"
- **KL 散度公式？交叉熵与 KL 散度的区别？**
  - KL(P∥Q) = Σ P(x)·log(P(x)/Q(x))；交叉熵 H(P,Q) = -Σ P(x)·log Q(x) = H(P) + KL(P∥Q)。交叉熵包含真实分布自身的信息熵，KL 散度只衡量分布的"额外差异"。在 RL 中 KL 散度用于约束策略更新不要偏离 ref model 太远
- **交叉熵公式？负对数似然？**
  - 交叉熵 L = -Σ y·log(ŷ)，本质上就是负对数似然——取对数后取负，最小化交叉熵等价于最大化似然
- **DPO 原理？**
  - DPO（Direct Preference Optimization）直接从偏好对中学习，无需显式训练 Reward Model。它将 Bradley-Terry 偏好模型中的 reward 替换为策略对数概率比，直接在偏好数据上优化策略，避免了 PPO 的两阶段训练
- **Reward Model 的训练方式？**
  - 通常使用偏好对数据（chosen vs rejected response），用 Bradley-Terry 模型建模偏好概率，通过对比损失（contrastive loss）训练模型给 chosen 更高分、rejected 更低分
- **多任务学习如何缓解数据不平衡？**
  - 可以通过动态损失加权（如 Uncertainty Weighting）、梯度归一化（GradNorm）、或对不同任务进行过采样/欠采样来平衡各任务对梯度的贡献
- **不除以 √dₖ 会出现梯度消失还是爆炸？**
  - 不除以 √dₖ 会导致点积值过大，经过 Softmax 后分布极度尖锐（接近 one-hot），梯度趋近于零——即梯度消失。除以 √dₖ 将方差控制在 1，保持 Softmax 输出的平滑性和梯度的有效性

### 项目一 · 搜索策略部门 — query 拆解压缩业务
**面试时间**：2025 年 4 月 25 日（一面）、4 月 25 日（二面）

**一面问题**：
- **LoRA 显存反而变大，为什么？从优化器角度、梯度角度分析？**
  - LoRA 虽然减少了可训练参数量，但 Adam 优化器仍需为 LoRA 参数维护一阶和二阶动量，如果 LoRA 的 rank 较大或适配模块较多，优化器状态的开销可能超过全参微调时节省的梯度存储；此外前向激活仍与全参微调相同
- **PPO 的 Reward Model 和 Value Model 分别计算什么？（客服场景）**
  - Reward Model 对完整 response 打分，评估回复质量（如礼貌度、解决率）；Value Model（Critic）估计当前状态（已生成的部分序列）的期望累积回报，用于计算 Advantage = R - V(s)，指导每个 token 的更新方向
- **PPO 的 Value Model 和 Reward Model 是针对 token 还是 response 进行奖励？**
  - Reward Model 通常对整个 response 打分（outcome reward），Value Model 则对每个 token 位置估计未来期望回报，从而实现逐 token 的细粒度优化
- **算法题**：最长连续序列

**二面问题**：
- **推理压缩 / CoT 压缩？**
  - 如何在不显著损失推理质量的前提下压缩思维链长度？常见方法包括：蒸馏短 CoT、自适应早停、只保留关键推理步骤的摘要
- **GRPO 采样方法？**
  - GRPO 对每个 prompt 采样多条 response，在组内计算相对优势（用组内均值/标准差归一化），无需 Value Model，直接用 Reward 的组内相对排序作为优化信号
- **R1 训练过程？**
  - DeepSeek-R1 先通过大规模冷启动 SFT 构建基础能力，再使用 GRPO 进行强化学习（仅用 accuracy/format reward），通过训练过程中自然涌现的 Long-CoT 来提升推理能力
- **LoRA 原理？**
  - LoRA 在预训练权重旁添加低秩分解矩阵（A·B），仅训练 A 和 B，W' = W + α·A·B。推理时可合并回原权重，无额外推理开销
- **算法题**：搜索二叉树中第 K 小的元素

## 2025 年 5 月

### 虎牙 · 算法岗（AI 赋能直播方向）
**面试时间**：2025 年 5 月 27 日

**面试问题**：
1. **Reranker 与 Embedding 的区别？**
   - Embedding 模型将文本映射为固定维度的向量，适合大规模召回（向量检索速度快）；Reranker 对 query-doc 对做精细化的相关性打分（通常用 Cross-Encoder），精度更高但速度慢，一般用于召回后的精排阶段
2. **Embedding 如何训练？**
   - 通常使用对比学习（Contrastive Learning），正样本对拉近、负样本对推远。训练数据可来自点击日志（点击=正样本）、人工标注、或利用 LLM 合成。常见框架如 BGE 使用 InfoNCE 损失 + 难负样本挖掘
3. **算法题**：寻找数组中第 K 大的数（快速选择 QuickSelect）
4. **DeepSeek 训练过程？GRPO 相比 PPO 的改进与优势？**
   - DeepSeek-R1 采用 SFT → GRPO 的两阶段范式。GRPO 相比 PPO 的核心优势：去掉了 Critic（Value Model），用组内相对比较代替绝对打分，大幅降低显存和训练复杂度；不需要 reward model 与 value model 对齐，避免了 reward hacking 中的 reward-model 过拟合
5. **场景题**：直播助手——如何实现能生成符合直播风格回复弹幕的大模型？
   - 思路：收集直播间真实弹幕数据做 SFT，构建直播风格语料库（口语化、网络用语、梗文化）；设计风格 Reward（活泼度、互动感）做 GRPO 对齐；可引入 RAG 检索当前直播上下文（主播说了什么、弹幕氛围）作为 prompt 输入

**算力资源**：约 10 台 A 卡

### 智谱 AI · ToB 方向 · 算法岗
**面试时间**：2025 年 5 月 30 日

**面试问题**：
- **Long-CoT 训练时是否携带 CoT 历史信息？**
  - 视任务而定——如果是多轮对话场景，需要携带历史的 think & response 以保持上下文连贯性；如果是单轮推理任务，每次独立生成 CoT 即可。携带历史会增加序列长度和训练成本，但能提升多轮一致性
- **如何评估蒸馏 Long-CoT 的质量？**
  - 可从多维度评估：① Self-Consistency（多次采样答案一致性）；② Majority Vote 准确率；③ CoT 步骤的逻辑连贯性（可用 LLM-as-Judge 评估）；④ 最终答案正确率。TODO：参考 agent eval 相关论文，自动化评估框架
- **核心观点**：SFT 决定下限，GRPO 决定上限——应先分析 SFT 的 bad case，把 SFT 训练好再进入 GRPO 阶段
- **GRPO 训练曲线分析：Reward 下降的原因是什么？熵崩塌？如何解决？**
  - Reward 下降可能原因：① 模型已收敛到局部最优，探索不足；② KL 约束过强，策略无法优化；③ Reward 信号本身有噪声。熵崩塌指模型输出分布变得过于确定（接近 one-hot），失去多样性。解决方案：增大 KL 惩罚系数、提高采样温度、引入熵正则项、使用 DAPO 的 clip higher
- **GRPO 中 ref model 的 KL 散度系数变化的影响？**
  - KL 系数控制策略更新幅度——太小则策略可能偏离过远导致模式崩塌（mode collapse）；太大则优化受限，模型难以提升。理想做法是动态调整，可引申 DAPO 中 clip higher 的策略：对高 reward 样本不施加 KL 惩罚，鼓励探索优质方向
- **（简历中"推理速度"关键词触发）引申出 GQA、KV Cache**
  - GQA（Grouped Query Attention）将多个 Query Head 共享同一组 KV Head，在 MHA 和 MQA 之间折中，减少 KV Cache 显存同时保持质量；KV Cache 缓存已计算的 Key/Value，避免重复计算，是推理加速的核心技术
- **Qwen3 使用哪种 Attention？**
  - Qwen3 使用 GQA（Grouped Query Attention），平衡了推理效率和模型质量
- **SFT 的 bad case 分析方法与优化策略？**
  - 按错误类型分类（事实错误、逻辑错误、格式错误等），分析共性；针对性优化训练数据（补充同类样本、提升数据质量）；也可用 GRPO 在特定 bad case 上做针对性强化
- **GRPO 数据量、训练时长？**
  - 数据量通常在几千到几万条 prompt 级别，每条 prompt 采样 4-16 条 response；训练时长取决于模型规模和硬件，7B 模型通常几小时到一天
- **MoE 与 Dense 模型的优劣对比？不同场景如何选型？**
  - MoE 优势：相同计算量下参数量更大，知识容量更高；劣势：训练不稳定、推理显存高（需加载所有专家）、通信开销大。选型：资源充裕且追求极致性能选 MoE；追求部署效率和稳定性选 Dense
- **RAG 场景下 Embedding 模型如何训练？**
  - 使用领域内 query-doc 对做对比学习；可引入难负样本（hard negative）提升区分能力；使用 Matryoshka 表示学习支持可变维度；在目标领域数据上做 domain adaptation 微调

  - 考察对推理延迟的敏感度——首 token 延迟和总生成时间，如何优化（如调整 max_tokens、使用投机解码、KV Cache 优化等）
- **GRPO 原理？**
  - GRPO 对每个 prompt 采样一组 response，在组内用 Reward 的相对排名（归一化后的 advantage）作为优化信号，无需 Value Model，训练更稳定、显存更省
- **MoE 对比 Dense 模型的优势与劣势？**
  - MoE 用稀疏激活实现大参数量但低计算量；优势是知识容量大、相同 FLOPs 下性能更强；劣势是显存占用高（需加载所有专家）、通信开销大、训练不稳定、推理部署复杂
- **Tokenizer 算法？**
  - 主流是 BPE（Byte-Pair Encoding），通过统计字符对频率迭代合并，构建子词词表；还有 SentencePiece、WordPiece 等变体。BPE 的优势是能处理 OOV 问题
- **GQA 原理？**
  - Grouped Query Attention：将多个 Query Head 分为一组，共享同一对 KV Head。相比 MHA 减少 KV Cache 大小，相比 MQA 保持更好质量
- **LoRA 推理速度？**
  - LoRA 推理时可以将 A·B 合并到原始权重 W' = W + αAB，因此推理速度与全参模型完全相同，无额外开销
- **LoRA 推理与全参推理显存占用对比？**
  - 合并后显存完全一致——模型权重是同样大小的矩阵。LoRA 节省的是训练时的显存（优化器状态 + 梯度），而非推理时的显存
- **vLLM 原理？**
  - vLLM 的核心是 PagedAttention——将 KV Cache 以"页"为单位管理，类似操作系统的虚拟内存分页，避免显存碎片，大幅提升吞吐量；还支持 Continuous Batching，动态插入新请求

### 快手 · 支付商业 · 内容质量部门 · 算法岗
**面试时间**：2025 年 7 月 19 日

**一面问题**：
1. **针对测试集中的 test case，应该如何优化算法？**
   - 不应针对特定 test case 做硬编码优化（过拟合测试集）。正确做法：分析 test case 暴露的模型能力短板，回溯训练数据的覆盖度，针对性地补充训练数据或调整训练策略
2. **将有序链表转换为二叉平衡搜索树**
   - 经典的分治+快慢指针题。核心：用快慢指针找链表中点作为根节点，递归构建左右子树。时间复杂度 O(nlogn)（快慢指针）或 O(n)（先转数组再分治）

**算法题解法**：
```python
# 快慢指针 + 链表断开
def sortedListToBST(head: ListNode) -> TreeNode:
    if not head:
        return None
    if not head.next:  # 单节点直接返回
        return TreeNode(head.val)

    # 快慢指针找中点，prev 记录中点前驱（用于断开链表）
    prev, slow, fast = None, head, head
    while fast and fast.next:
        prev = slow
        slow = slow.next
        fast = fast.next.next

    prev.next = None  # 断开链表

    root = TreeNode(slow.val)
    root.left = sortedListToBST(head)
    root.right = sortedListToBST(slow.next)

    return root
```

### 网易有道 · 算法岗
**面试时间**：2025 年 7 月 22 日（一面）、7 月 24 日（二面，挂）

**一面问题**：
- **算法题**：最少硬币数量（DP）——给定面额数组和目标金额，求凑成目标的最少硬币数，经典完全背包变体
- 围绕项目三实习项目展开：
  1. **GRPO 和 SFT 训练的 Loss 分别是什么？**
     - SFT 使用交叉熵损失（逐 token 与标签对比）；GRPO 使用策略梯度损失 = clip(π/π_old)·Advantage - β·KL(π∥π_ref)，其中 Advantage 是组内 Reward 的归一化值
  2. **GRPO 数据整备与课程学习的原因？**
     - 课程学习从简单样本到困难样本逐步训练，有助于稳定训练、避免模型一开始就面对极端 case 导致崩溃
  3. **SFT 训练比 Generate 快的原因？损失函数？**
     - SFT 使用 Teacher Forcing——所有 token 并行计算损失（已知正确答案），只需一次前向传播；Generate 是自回归逐 token 生成，每步依赖前一步输出，无法并行
  4. **GRPO 超参数及其影响？**
     - 关键超参数：KL 系数（控制策略更新幅度）、采样数（每条 prompt 采几条 response）、学习率、clip 范围（控制策略比率裁剪）、max_length。KL 系数太小会 reward hacking，太大则训练停滞
  5. **Generate 参数：top-p、top-k、temperature？**
     - temperature 控制分布的尖锐度（>1 更随机，<1 更确定）；top-k 只保留概率最高的 k 个 token；top-p（nucleus sampling）保留累积概率达到 p 的最小 token 集合

**二面问题**：
- **Long-CoT 如何生成？**
  - 通过精心设计的 prompt 引导模型在 `think` 标签内进行逐步推理，再在 `response` 标签内给出最终回答。数据构造时用强模型（如 DeepSeek-R1）生成带推理链的回复，蒸馏给目标模型
- **GRPO 数据慢的原因？**
  - GRPO 需要在每个训练步对每个 prompt 采样多条完整 response（Rollout），生成过程是自回归的，且需要等所有 response 生成完才能计算 Reward，I/O 瓶颈严重
- **GRPO：batch 大小？采样规模？**
  - 一般 batch 为几十到几百条 prompt，每条 prompt 采样 4-16 条 response。总样本量 = prompt 数 × 采样数，需要在多样性和计算效率间平衡
- **TRL 设置的最大生成长度？**
  - TRL 中通过 `max_new_tokens` 或 `max_length` 控制，需根据任务设置——太短截断有效内容，太长浪费算力且影响训练速度
- **Loss 起始熵值？Loss 到什么时候不再变化？**
  - 初始 Loss 接近随机策略的熵（ln(vocab_size) ≈ 10-12 for 30k+ vocab）；当 Loss 连续几个 epoch 不再下降或验证集指标不再提升时考虑停止
- **Reward 起始值与最终值？**
  - 起始 Reward 通常较低（模型未经对齐），随着 GRPO 训练逐步提升。需关注提升的绝对值——从多少提升到多少，以及提升是否来自真正的质量改善而非 reward hacking
- **GRPO 过程中响应长度如何变化？**
  - 通常响应长度会随着训练逐渐增长（模型学会展开推理），但如果 Reward 中有 length penalty，会控制长度在合理范围
- **TRL Rollout 的方法？**
  - TRL 使用 `generate()` 方法进行 Rollout——对每个 prompt 批量生成多条 response，支持 vLLM 加速生成。Rollout 阶段使用当前策略模型（actor），生成后计算 Reward 再进行梯度更新

- **SFT 后为什么还需要 GRPO？**
  - SFT 让模型"会说话"，GRPO 让模型"说得好"。SFT 只能学到数据中的平均水平，GRPO 通过奖励信号引导模型探索更好的回复策略，突破 SFT 的上限。典型例子：SFT 后模型能完成基本对话，但缺乏主动引导、情绪安抚等高级能力，GRPO 可针对性强化
- **PPO 与 GRPO 八股，区别是什么？**
  - PPO 需要四个模型（Actor、Critic、Ref、Reward），GRPO 只需三个（Actor、Ref、Reward），去掉了 Critic。PPO 用 GAE 估计 Advantage，GRPO 用组内 Reward 归一化。GRPO 更简单、更省显存，在 LLM 对齐任务中表现不输 PPO
- **开源模型结构介绍：Qwen、Bloom**
  - Qwen 系列：Decoder-only Transformer，使用 GQA、RoPE、SwiGLU 激活，支持多语言；Bloom：也是 Decoder-only，使用 ALiBi 位置编码（非 RoPE），在 ROOTS 多语言语料上训练
- **Manus 结构？**
  - Manus 是一个通用 AI Agent 框架，采用多 Agent 协作架构，通过规划-执行-验证的循环来完成复杂任务。核心是任务分解 + 工具调用 + 结果验证
- **二分类评估指标：F1、Precision、Accuracy 如何计算？F1 公式为什么如此构造？系数含义？**
  - F1 = 2PR/(P+R) = 1/[(1/P + 1/R)/2]，调和平均数——只要 Precision 或 Recall 有一个低，F1 就低。调和平均比算术平均对低值更敏感，适合对两者均有要求的场景。召回率低 → FN 过多（漏判）；精确率低 → FP 过多（预测不准）
- **为什么需要 CoT？**
  - CoT 将复杂推理分解为多步中间步骤，每步更简单、更可验证；显式的推理过程提升了答案的准确性和可解释性；在数学、代码等需要多步推理的任务上提升显著
- **Encoder-only 与 Decoder-only 模型架构的区别？Mask 的区别？**
  - Encoder-only（如 BERT）：双向注意力（能看到前后所有 token），适合理解任务；Decoder-only（如 GPT）：因果注意力（causal mask，只能看到前面的 token），适合生成任务。Encoder-Decoder（如 T5）：Encoder 双向 + Decoder 单向（cross-attention 可看全部 Encoder 输出）
- **根据 hidden_size 估算显存占用量？**
  - 模型参数：参数量 × 2 bytes（FP16）/ × 4 bytes（FP32）；KV Cache：(2 × n_layers × n_kv_heads × head_dim) × batch_size × seq_len × 2 bytes；优化器状态：参数量 × 8 bytes（Adam FP32）/ × 12 bytes（Adam + momentum）；总显存 ≈ 参数 + 梯度 + 优化器 + 激活 + KV Cache

  - MLA 在低维潜在空间做 RoPE（对 query 和 key 的潜在表示分别施加旋转），而非在展开后的高维空间做，这样既保留了位置信息又维持了压缩效率

**二面问题**：
- **专家并行、Pipeline 并行、Tensor 并行的区别？**
  - 专家并行（EP）：MoE 中不同专家分布在不同设备，按路由结果分发 token；Pipeline 并行（PP）：按层切分，不同设备负责不同层，用微批次流水线减少气泡；Tensor 并行（TP）：单层内按矩阵维度切分（行/列切分），通信量最大但适合超大规模模型
- **BGE 优化？**
  - BGE（BAAI General Embedding）使用 RetroMAE 预训练 + 对比学习微调，关键优化包括：难负样本挖掘、指令微调提升泛化、Matryoshka 表示支持可变维度
- **MLA、GQA、MHA 的区别？**
  - MHA 每个 Head 独立 QKV；GQA 多个 Q Head 共享一组 KV；MLA 将 KV 压缩到低维潜在空间，在潜在空间做注意力，进一步减少 KV Cache
- **位置编码？**
  - 绝对位置编码（Sinusoidal / Learned）vs 相对位置编码（RoPE / ALiBi）。RoPE 是目前主流——通过旋转将相对位置信息编码到注意力分数中，外推性好
- **MLA 的旋转位置编码与普通 RoPE 有什么不同？**
  - 普通 RoPE 在高维 Q/K 空间做旋转；MLA 在压缩后的低维潜在空间做 RoPE，需要特殊设计以保证位置信息在压缩/解压过程中不丢失
- **Embedding 模型如何训练（以 BGE 为例）？**

  - GRPO 训练中如果没有 length penalty，模型倾向于生成更长回复——因为更长的推理链通常能获得更高 Reward（思考更充分）。这是 RL 训练中的常见现象，需通过 length reward 或 max_length 控制

  - SFT 的 Softmax + 交叉熵只能做"正向引导"——告诉模型正确答案是什么，无法对错误行为施加显式惩罚。RL 的 Reward 可以是负数，明确告诉模型"这样做不好"。SFT 是"学好的"，RL 是"学好且避开坏的"

### CVTE 视源股份 · 算法岗（教育 / 会议方向）
**面试时间**：2026 年 1 月 22 日（一面）、2 月 3 日（二面）、2 月 12 日（HR 面）

**一面问题**：
- **Beam Search 原理？**
  - 束搜索——每步保留 top-k 个最优候选序列（k 为 beam size），而非仅保留最优一个（贪心）或所有可能（穷举）。在生成质量和计算效率之间折中，k 越大质量越好但越慢
- **模型推理能否保证完全一致性？**
  - 不能——由于采样的随机性（temperature、top-p/k）和浮点运算的非确定性，即使相同输入也可能产生不同输出。可通过设置 temperature=0 和固定随机种子来实现确定性输出
- **旋转位置编码（RoPE）原理？**
  - RoPE 通过对 Q 和 K 向量施加旋转变换来编码位置信息——旋转角度随位置递增，使得 Q·K 的内积自然地包含相对位置信息。优势：外推性好、不增加参数量、与 Attention 机制无缝融合
- **推理加速方法：投机解码**
  - 用小模型快速生成候选 token 序列，大模型一次验证多个 token（通过一次前向传播并行检查所有候选 token 的概率），验证通过的 token 直接采纳。本质是用小模型的速度 + 大模型的质量

### 字节跳动 · 番茄小说 · 算法岗（搜广推 / 大模型方向）
**面试时间**：2026 年 1 月 26 日（一面）

**一面问题**：
- **模型训练效果越来越好，但 Reward 不再适用时如何处理？**
  - 这是 Reward Hacking 的典型场景——模型已超越 Reward Model 的判别能力。解决方案：① 升级 Reward Model（用更强模型标注）；② 引入更难的任务（课程学习）；③ 使用对抗训练；④ 引入人工评估作为最终标准
- **训练不稳定或崩溃的原因及解决方案？**
  - 原因：学习率过大、梯度爆炸、数据质量问题、KL 约束不当。解决：降低学习率、梯度裁剪、warmup、检查数据质量、调整 KL 系数
- **训练中出现"复读机"现象如何解决？**
  - 原因：模型陷入局部最优（重复输出高概率序列）。解决：提高温度、降低 top-p、加入重复惩罚（repetition penalty）、在 Reward 中惩罚重复
- **过拟合如何应对？**
  - 正则化（Dropout、Weight Decay）、数据增强、早停、增大数据量、降低模型复杂度
- **DeepSpeed 三个阶段的区别？**
  - Stage 1：分片优化器状态（8× 显存节省）；Stage 2：分片优化器 + 梯度；Stage 3：分片优化器 + 梯度 + 参数（最省显存但通信最多）
- **LoRA 参数一般取多少？**
  - rank r 通常取 8、16、32、64；alpha 通常为 r 的 1-2 倍。r=16 是常见默认值，越大可学习参数越多但显存也越大
- **蒸馏有哪些方式？**
  - 数据蒸馏（用大模型生成训练数据给小模型）、知识蒸馏（匹配 logits 或中间层特征）、上下文蒸馏（将推理链从大模型迁移到小模型）
- **AutoPE 是什么？**
  - 自动 Prompt Engineering——利用模型自动搜索和优化 prompt，替代人工调 prompt
- **Flash-Attention 原理？**
  - 通过分块（Tiling）和重计算（Recomputation）技术，将 Attention 的显存复杂度从 O(n²) 降到 O(n)，同时利用 GPU SRAM 加速 IO，实现更快更省显存的 Attention 计算
- **KL 散度与交叉熵的区别？**
  - 交叉熵 = 熵 + KL 散度。SFT 最小化交叉熵（让模型输出逼近标签）；RL 中 KL 散度作为约束（限制策略不要偏离 ref model 太远）

**算法题**：
1. 接雨水（双指针/单调栈）
2. 单头 Attention 实现

**二面问题**：
- **Reward Hacking 问题：group 内样本都变得很好时有什么解决方案？**
  - 引入更难的评估维度、使用对抗样本、动态调整 Reward 标准、引入多样性奖励（鼓励不同风格的优质回复）
- **DAPO 能否解决该问题？**
  - DAPO 的 clip higher 可以让高 Reward 样本不受 KL 约束，鼓励探索更多样化的优质策略，一定程度上缓解 reward hacking
- **残差网络中信号放大导致深层崩溃，如何应对？**
  - 使用 Pre-Norm（先归一化再 Attention/FFN）替代 Post-Norm、使用更好的初始化策略（如 DeepNorm）、使用 scaled residual（对残差分支乘以小于 1 的系数）
- **哈希表底层实现原理？**
  - 数组 + 链表/红黑树——通过哈希函数映射到数组索引，冲突时用链地址法（链表）或开放地址法解决。Java HashMap 在链表长度 > 8 时转为红黑树

- LeetCode 380/381：O(1) 时间插入、删除、随机获取——用哈希表 + 动态数组，删除时用数组末尾元素替换被删元素
