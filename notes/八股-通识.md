## 词袋模型与 Word2Vec

### Word2Vec：CBOW & Skip-gram

**灵感来源**：一个词周围词语的语义和当前词的语义是类似关联的，因此可以用周围词来预测当前词，反之亦然。

Word2Vec 包含两种模式：
1. **CBOW（Continuous Bag-of-Words）**：用上下文词预测中心词
2. **Skip-gram**：用中心词预测上下文词

**CBOW 训练过程**：
- 包含两个线性层：输入嵌入矩阵和输出嵌入矩阵
- 将一个词的前后 n 个词通过输入嵌入矩阵映射为向量后做池化（通常是平均池化）
- 将该池化向量与输出嵌入矩阵相乘，得到词表大小的概率分布
- 用 one-hot 形式的 Ground Truth 与该概率分布计算交叉熵损失

**目标函数**：
```
CBOW：      L = Σ log P(w_t | w_{t-n}, ..., w_{t+n})
Skip-gram：  L = Σ Σ log P(w_{t+j} | w_t)
```

## LSTM / RNN / CNN

### RNN（循环神经网络）

**核心思想**：每个时间步的隐藏状态不仅取决于当前输入，还取决于上一时间步的隐藏状态，从而实现"记忆"。

**公式**：
```
h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b_h)
y_t = W_hy · h_t + b_y
```

**主要问题**：梯度消失/梯度爆炸、长距离依赖能力弱。

### LSTM（长短期记忆网络）

**核心改进**：引入门控机制和细胞状态（Cell State），有效缓解 RNN 的长距离依赖问题。

**三个门控**：

1. **遗忘门**：决定从细胞状态中丢弃哪些旧信息
   ```
   f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
   ```
2. **输入门**：决定将哪些新信息存入细胞状态
   ```
   i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
   C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)
   ```
3. **输出门**：决定细胞状态的哪些部分输出到隐藏状态
   ```
   o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
   h_t = o_t * tanh(C_t)
   ```

**细胞状态更新**：`C_t = f_t * C_{t-1} + i_t * C̃_t`

**形象理解**：遗忘门 = "忘掉哪些旧知识"，输入门 = "学哪些新知识"，输出门 = "用到哪些知识"

### CNN（卷积神经网络）

**核心思想**：通过卷积核滑动提取局部特征，具有权值共享（参数少）和局部感受野（关注邻域）的特性。

**关键组件**：卷积层（提取局部特征）、池化层（降维、保留主要特征）、全连接层（映射到输出空间）

**在 NLP 中的应用**：用不同尺寸的卷积核（如 kernel_size=3,4,5）提取 n-gram 特征。优点：并行计算能力强、局部特征提取高效；缺点：感受野受限于卷积核大小，难以捕捉长距离依赖（相比 Transformer）。

## Tokenizer 分词器

### 常见分词器类型

**1. BPE（Byte Pair Encoding）**：迭代合并出现频率最高的相邻符号对，逐步构建子词词表。步骤：初始词表 = 所有单个字符 → 统计相邻符号对频率 → 合并频率最高的 → 重复直到达到预设大小。优点：在词表大小和分词粒度间取得良好平衡，能有效处理未登录词（OOV）。

**2. WordPiece**：BERT 采用的分词方式。与 BPE 类似，但基于语言模型概率（likelihood gain）而非原始频率来决定合并顺序。

**3. Unigram Language Model**：从一个较大的种子词表开始，迭代删除使困惑度增加最少的子词，直到词表缩小到目标大小。SentencePiece 中常用。

### 投机采样（Speculative Decoding）与 EAGLE

**核心思想**：用一个轻量草稿模型快速生成若干候选 token，再由大模型一次性并行验证所有候选。在不损失精度的前提下，大幅减少大模型的串行推理次数。

**EAGLE（Extrapolation-Aware Generation with speculative decoding）的改进**：
- 将草稿模型从独立的小模型替换为大模型前几层的特征提取器，节省额外显存
- 利用大模型的特征图（而非独立生成）来预测候选 token，实现更高效的速度-质量 trade-off

**Byte ↔ Token 换算**：1 token ≈ 0.75 个英文单词，1 token ≈ 1.3 个中文字符。理解此比例有助于推理速度和显存的快速估算。

## Transformer

### 为什么 Scaled Dot-Product Attention 需要除以 √d_k

原始的 Q 和 K 假设为均值为 0、方差为 1 的独立随机变量。Q 与 K 的内积是 d_k 个独立乘积项之和：

```
Var(Q · K^T) = d_k
```

随着 d_k 增大，内积的方差线性增长。这会导致 softmax 的输入值过大，输出趋近于 one-hot 分布（某些位置接近 1，其余接近 0），进而导致梯度趋近于零（梯度消失）。除以 √d_k 将方差恢复为 1：

```
Var((Q · K^T) / √d_k) = 1
```

这使得 softmax 的输出更加平缓，保证梯度正常回传，避免梯度消失。

### 残差连接为什么能防止梯度消失

**残差连接公式**：
```
y = F(x, {W_i}) + x
```

对 x 求偏导时，shortcut 路径贡献的导数恒为 **1**，不受非线性激活函数导数（通常 ≤1）的影响：

```
∂y/∂x = ∂F/∂x + 1
```

即使 ∂F/∂x 很小（接近 0），梯度仍至少有 1 的量级。**残差连接始终提供一条梯度为 1 的直通路径，防止梯度消失。**

> 注意：防止梯度消失和爆炸是由残差连接和归一化层**共同作用**的：
> - 归一化层（LayerNorm/RMSNorm）将值控制在合理范围（均值为 0、方差为 1），防止前向传播时数值爆炸
> - 残差连接确保反向传播时梯度不衰减（shortcut 始终贡献 1）

### Pre-LN 与 Post-LN

**位置区别**：
- **Post-LN**：LayerNorm 放在残差连接之后（Transformer 原始论文）
- **Pre-LN**：LayerNorm 放在注意力/FFN 之前

**性能对比**：

| 特性 | Post-LN | Pre-LN |
|------|---------|--------|
| LayerNorm 位置 | 残差连接之后 | 注意力/FFN 之前 |
| 训练稳定性 | 需要 warm-up，深层易梯度爆炸 | 更稳定，无需 warm-up |
| 收敛速度 | 较慢 | 较快 |
| 最终性能 | 通常略好 | 稍差但差距不大 |
| 梯度流 | 梯度范数随层数指数增长 | 梯度范数保持稳定 |

**Pre-LN 更稳定的原因**：将每个子层的输入提前标准化，避免了 Post-LN 中残差分支累积导致的梯度爆炸问题。当前主流大模型（GPT、Llama 等）普遍采用 Pre-LN。

### Softmax 函数

**为什么需要用 e 为底数？** 上一层输出（logits）有正有负，指数函数将其全部映射为正数，才能计算概率分布。同时，e^x 的导数等于自身，配合交叉熵损失时求导形式特别简洁（梯度 = softmax(x) - y）。

**防 NaN 版本（数值稳定实现）**：
如果 logits 中存在较大值（如 100），e^100 会上溢为 inf，导致 NaN。一种简单有效的解决方案是让所有输入平移到 0 附近：令 F = -max(a1, a2, ..., an)，分子分母同乘以 e^F，相当于将所有输入平移。减去最大值后所有输入 ≤0，e^x ∈ (0,1]，避免上溢。除了最大值，其他输入值都被平移成负数，e 为底的指数函数越小越接近 0，这种方式比得到 NaN 的结果更好。

**代码**：
```python
# 基础版
def softmax(x):
    exp_x = np.exp(x)
    return exp_x / np.sum(exp_x)

# 防 NaN 版
def softmax(x):
    shift_x = x - np.max(x, axis=-1, keepdims=True)
    exp_x = np.exp(shift_x)
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)
```

## 位置编码

### 绝对位置编码：正弦位置编码（Sinusoidal）

**加入位置**：在 embedding 层与 token embedding **相加**，只加一次。相当于把位置信息注入到 token 中。

**原理**：两两一组使用 sin(ω_k · pos) 和 cos(ω_k · pos)，其中 ω_k = 10000^{-2k/d}。

```
PE(pos, 2i)   = sin(pos / 10000^{2i/d})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d})
```

- **高频成分**（大角频率 ω_k，小周期）→ 捕捉局部/短距离 token 信息
- **低频成分**（小角频率 ω_k，大周期）→ 捕捉长距离 token 信息
- 不同频率能激活不同注意力头（multi-head attention）：某些头专门响应高频捕捉局部词序，某些头响应低频捕捉长程依赖

**正交性**：不同频率组之间内积为零，互不干扰——因为不同周期的正余弦在一个周期内的内积是正交的。

**局限性**：
- **非结构性**：加法形式导致点积无法结构化反映相对距离 i-j 的影响
- **静态性**：位置编码固定不变，不随输入或训练而改变，模型需被动适应

### 相对位置编码：RoPE（旋转位置编码）

**加入位置**：与绝对位置编码不同，RoPE 作用在**每一层的 Q 和 K 向量上**。

**原理**：通过二维旋转矩阵将位置信息编码到 Q 和 K 向量中。对于位置 m 的向量 q_m 和位置 n 的向量 k_n，RoPE 使得它们的点积仅依赖于相对位置 (m-n)：

```
(Q_m · R_m)(K_n · R_n)^T = Q_m · R_{m-n} · K_n^T
```

**多尺度设计**：
- 组别越小（维度越低），角速率 θ 越大，转动角度越大 → 高频，捕捉局部关系
- 组别越大（维度越高），角速率 θ 越小，转动角度越小 → 低频，捕捉长上下文关系

**实现代码**：
```python
import torch

def rope(x):
    """旋转位置编码实现"""
    batch_size, seq_len, head_dim = x.shape
    device = x.device

    # theta = 10000^{-2i/d}，频率随维度递减
    theta = 10000 ** (-torch.arange(0, head_dim, 2, device=device) / head_dim)
    pos = torch.arange(seq_len, device=device).unsqueeze(1)
    angles = pos * theta  # [seq_len, head_dim/2]

    cos, sin = torch.cos(angles), torch.sin(angles)

    # 每两个维度为一组做二维旋转
    x1, x2 = x[..., 0::2], x[..., 1::2]
    rotated_x1 = x1 * cos - x2 * sin
    rotated_x2 = x1 * sin + x2 * cos

    return torch.stack([rotated_x1, rotated_x2], dim=-1).flatten(-2)
```

### MLA 中的 RoPE（DeepSeek V2）

**MLA（Multi-head Latent Attention）核心思想**：将 KV 向量通过低秩矩阵压缩为低维潜在向量（c_kv），大幅减少 KV Cache 显存。

**低秩压缩与 RoPE 的直接冲突**：

1. **位置信息与压缩路径耦合**：RoPE 的旋转矩阵 e^{imθ} 与升维矩阵 W^{UK} 混合，导致位置编码被低秩投影破坏
2. **推理时无法合并计算**：由于 RoPE 的旋转矩阵 e^{imθ} 是位置相关的（非固定），而 W^{UK} 是固定参数，二者无法在推理时合并为单一线性变换（矩阵乘法不满足交换律）。因此，每次生成新 token 时需要重新计算所有历史 Key 的 W^{UK} · c_kv 再应用 RoPE，导致推理速度大幅下降。

**解决方案**：把 Key 拆成两份：
- **k^C**：不含位置信息的"内容"向量，可以低秩压缩
- **k^R**：只用极少维度（如 64 维）单独做 RoPE，保存位置信息

这样 KV-Cache 只存 k^C 的低秩表示 + k^R 的小矩阵，显存降到原来的 40%~60%。

### ALIBI（Attention with Linear Biases）

**核心思想**：不学习位置编码，而是通过一个固定的下三角距离矩阵，为每个 attention head 施加不同斜率的线性衰减偏置，鼓励模型更多地关注近处的 token。

**公式**：
```
Attention(Q, K, V) = softmax(QK^T / √d_k + m · B) V
```
其中 B 是下三角距离矩阵（B_{ij} = -(i-j) for i ≥ j），m 是每个 head 的可预设斜率。

**关键特性**：
- 偏置是**不学习的**，全程由公式决定
- 所有 head 共用同一个距离矩阵模板，只是斜率不同（尺度不同）
- 以 head 为单位区分衰减斜率，让不同 head 感知不同的距离范围

## 注意力机制详解

### Self-Attention（SA）

Q、K、V 通过三个独立的线性变换从同一输入中投影得到，允许模型分别优化用于匹配（Q 和 K）和用于输出信息合成（V）的表示。这样做增加了模型的灵活性和表达能力。

**公式**：`Attention(Q, K, V) = softmax(QK^T / √d_k) V`

### Multi-Head Attention（MHA）

将 embed_dim 拆分为 num_heads × head_dim，每个头独立计算注意力后拼接。维度变化流程：
```
(bs, seq_len, hidden_dim)
    → reshape + transpose → (bs, num_heads, seq_len, head_dim)
    → QK^T/√head_dim + softmax → (bs, num_heads, seq_len, seq_len)
    → ×V → (bs, num_heads, seq_len, head_dim)
    → transpose + reshape → (bs, seq_len, hidden_dim)
```

**多头的作用**：
| 原因 | 解释 |
|------|------|
| 多视角建模 | 每个头关注不同的依赖关系或语义特征 |
| 多尺度建模 | 不同头对远距离与近距离关系建模能力不同 |
| 表达能力增强 | 多头拼接后经 W_o 线性组合，提供更丰富的组合信息 |
| 降低单头维度 | 每个头低维轻量，训练更容易收敛 |
| 结构先验（Inductive Bias） | 强化注意力机制捕获多种语言规律的能力 |

### Multi-Query Attention（MQA）

所有 Q 头共享同一对 K/V 投影矩阵（K/V 只有 1 个"头"），大幅减少 KV Cache 的显存占用，提升推理速度。MQA 被大量应用于 ChatGLM2 等 LLM 中。

**KV Cache 显存对比（32 层，head_dim=128，seq_len=2048，BF16）**：
- MHA（32 头）：32 × 32 × 128 × 2048 × 2 × 2B ≈ 128MB
- MQA（1 头）：32 × 1 × 128 × 2048 × 2 × 2B ≈ 4MB
- 节省约 96.9%

**代价**：K/V 的表达能力下降，可能带来轻微精度损失。

**MHA → MQA 转换方法**：将原始多个头的 K、V 投影矩阵进行均值池化，合并为单个投影矩阵，再进行少量步数的额外预训练以适配新结构。均值池化方法优于选择单个头或从头初始化。

### Grouped-Query Attention（GQA）

虽然 MQA 大幅减小了参数数量，但复杂度的降低可能导致质量下降和训练不稳定。GQA 将 Query 分组，每个组内共享一组 K/V，是 MHA 和 MQA 的折中方案。

```
MHA：每组 1 个 Q 头 → 组数 = 头数（如 32 组）
GQA：每组 n 个 Q 头 → 例如 64 头分 8 组，每组 8 个 Q 头共享一对 KV
MQA：所有 Q 头同组 → 组数 = 1
```

Llama 2 70B 使用 64 Q 头 + 8 KV 头（GQA-8）。GQA 在 Llama-2 和 Mistral 7B 中得到广泛应用。

### MLA（Multi-head Latent Attention）

DeepSeek V2 提出。将 KV 向量通过低秩矩阵压缩为低维潜在向量，大幅减少 KV Cache 显存（约 40%~60%）。详见[位置编码 - MLA 中的 RoPE](#mla-中的-rope-deepseek-v2)。

### NSA（Native Sparse Attention）

DeepSeek 提出的原生稀疏注意力机制。通过动态选择重要的 token 参与注意力计算，将稀疏性内建到注意力机制中（而非事后裁剪），在长序列场景下显著降低计算复杂度。

### MOBA

一种混合注意力架构，结合不同注意力机制（如局部窗口注意力 + 全局稀疏注意力）的优势，在效率与效果之间取得平衡。

### Causal Mask 与 Padding Mask

- **Causal Mask（因果掩码）**：上三角置 -inf，保证自回归生成时位置 i 只能看到 ≤i 的 token
- **Padding Mask**：将 [PAD] 位置的注意力权重置 -inf，忽略填充 token

### vLLM：PagedAttention

**核心问题**：传统 KV Cache 为每个序列分配连续的显存空间，产生大量内部碎片，显存利用率通常仅 20%~30%。

**PagedAttention 灵感**：来自操作系统的虚拟内存分页机制。

**工作原理**：
1. 将 KV Cache 划分为固定大小的 block（如 16 个 token）
2. 维护一个 block table 记录逻辑位置到物理块的映射
3. 物理块可以不连续存放，像 OS 的内存页一样管理

**优势**：
- 显存利用率从 20%~30% 提升至 90%+
- 支持更大的 batch size
- 吞吐量提升 2~4 倍
- Continuous Batching：请求完成立即调度下一个，无需等整个 batch
- Prefix Caching：共享相同前缀的 KV Cache

### Flash Attention

**V1 核心思想**：IO 感知（IO-aware）的注意力算法，减少 HBM（显存）和 SRAM（片上缓存）之间的数据搬运。

**传统注意力的瓶颈**：Q、K、V、S（注意力分数）、P（softmax 输出）都需要在 HBM 和 SRAM 之间反复搬运，标准实现需要 O(N²) 的 HBM 访问。

**Flash Attention 方案**：
- 分块计算（tiling）：每次只加载一个小块到 SRAM
- 重计算（recomputation）：前向不保存完整的 S 和 P，反向时重新计算
- 空间复杂度从 O(N²) 降至 O(N)，速度快 2~4 倍

**V2 改进**：更好的 thread block 调度策略、优化的 warp 同步、支持更大 head dimension。速度约为 V1 的 2 倍。

**V3 改进**：Hopper 架构优化（WGMMA 指令）、FP8 支持、更好的负载均衡。

### SDPA（PyTorch 融合注意力）

PyTorch 2.0+ 内置的 `F.scaled_dot_product_attention`，自动选择最优后端（Flash Attention / Memory Efficient Attention / CUDNN Attention），相比手动实现性能更优。

## 归一化

### ICS（内部协变量偏移）

**定义**：在深度神经网络训练过程中，**每一层的输入分布会因参数更新而发生动态变化**。当参数通过梯度下降更新时，前一层权重的调整会导致后一层的输入分布（均值、方差）持续改变。这种变化在深层网络中被逐层放大，使得高层需要不断适应底层的分布变化。

**ICS 带来的问题**：
1. **训练效率下降**：输入分布不稳定时，模型需要频繁调整学习率——学习率过大导致梯度爆炸，过小则收敛缓慢。高层网络需不断适应底层输入的分布变化，导致参数更新方向反复调整，延长收敛时间。
2. **梯度消失或爆炸**：输入分布变化可能使激活函数进入饱和区（如 Sigmoid 的极端值区域），导致梯度趋近于零（梯度消失）或数值溢出（梯度爆炸）。

归一化的作用正是缓解 ICS，使每层输入分布保持稳定，加速收敛。

### Batch Normalization（BN）

沿 batch 维度归一化——**对一批样本中同一个位置 token 的所有特征维度求均值和方差**。独立作用在每个特征上，对一个 batch 中所有 token 的同一个特征维度进行归一化。

**公式**：
```
μ_B = (1/m) Σ x_i
σ²_B = (1/m) Σ (x_i - μ_B)²
x̂_i = (x_i - μ_B) / √(σ²_B + ε)
y_i = γ · x̂_i + β
```

**为什么 BatchNorm 不适合 Transformer？**
1. **NLP 输入是变长序列，不适合 batch 的统计量**：BatchNorm 依赖于 batch 中同一维度上的统计量（均值和方差），但 NLP 中不同位置的 token 是有语义顺序的，把它们混在一起平均是不合适的
2. **BatchNorm 不支持 padding mask**：计算均值/方差时不能忽略 padding，会引入噪声；LayerNorm 则对每个 token 独立归一化，天然支持 masking
3. **在训练和推理之间切换不稳定**：BatchNorm 在训练时使用 batch 的均值和方差，推理时使用累积的全局均值与方差，这在 NLP 变长序列上非常不稳定
4. **动态上下文需求**：自注意力机制依赖位置间的动态交互，Batch Norm 的跨样本统计量可能破坏这种关系

**结论**：在 Transformer 中，Batch Norm 的归一化维度是**每个特征维度**，基于所有样本和位置的值。而 Layer Norm 更适合 Transformer 的动态输入特性。

### Layer Normalization（LN）

沿 hidden 维度归一化——**对每个样本中单个 token 的所有特征维度（如 768 个）独立计算均值和方差**，token 和 token 之间没有关系。若使用 Layer Norm，则每个 token 独立计算均值和方差，与其他 token 无关。

**公式**：
```
μ = (1/d) Σ x_j
σ² = (1/d) Σ (x_j - μ)²
x̂_j = (x_j - μ) / √(σ² + ε)
y_j = γ · x̂_j + β
```

**LN vs BN 面试简答**：Transformer 使用 LayerNorm 而不是 BatchNorm，主要因为：
- BN 依赖 batch size，在 NLP 里 batch 往往较小，推理时甚至是 1，统计量不稳定
- BN 跨样本、跨序列位置统计，会破坏 token 独立表征，不符合自注意力机制
- LN 只在 hidden 维度上做归一化，跟 batch size 无关，适合逐 token 的表示学习，推理时也更稳定

### RMSNorm

**公式**：
```
RMSNorm(x) = x / RMS(x) · γ
RMS(x) = √((1/D) Σ x_i²)
```

**特点**：不做均值减法（即没有去中心化），只保留尺度归一化 + 可学习的缩放参数 γ。通常**没有偏置 β**。

**优势**：
- **更快、更轻量**：省去了求均值和减均值的操作，计算更少，减少内存访问。从原本的三次遍历（算均值、算标准差、更新）变为两次遍历（只算均方根和更新）
- **对加速器更友好**（GPU/TPU），利于并行
- **更接近"缩放归一化"**：仅对模长进行归一化（长度标准化），保持了方向信息，不引入偏移，节省了参数

**LN vs RMSNorm 面试简答**：RMSNorm 是 LayerNorm 的简化版，它不减均值，只做尺度归一化。这样计算更快、更省参数，并且在超大规模 Transformer 中足够保证训练稳定性。LayerNorm 的均值中心化在大模型里不是必须的，反而会增加开销。实践表明，RMSNorm 在 LLM 中性能相当甚至更稳定，所以现在大模型普遍用 RMSNorm（Llama / Mistral / Qwen 等）。

### γ 和 β 参数的作用

γ（缩放）和 β（平移）都是模型的可学习参数。归一化后分布被限制在均值为 0、方差为 1，可能限制模型表达能力；γ 和 β 恢复了这一自由度，使模型能够**恢复表达能力**，避免归一化压平特征。Transformer 依赖这些参数来提高模型的稳定性，如果不设置这两个参数可能会影响模型中一些 Layer 的性能。PyTorch 的 `nn.LayerNorm` 自动为每个特征维度添加可训练的 γ 和 β。

## 激活函数

```python
import numpy as np
from scipy.stats import norm

def gelu(x):    return x * norm.cdf(x)                           # BERT/GPT 标准
def relu(x):    return np.maximum(0, x)                          # 简单高效
def swish(x, beta=1): return x * (1/(1 + np.exp(-beta * x)))     # β=1 即 SiLU
```

### Sigmoid

σ(x) = 1/(1+e^{-x})，输出 (0,1)。适合二分类输出层。缺点：梯度消失（两端饱和区导数趋近于 0）、输出非零中心。

### Tanh

tanh(x) = (e^x - e^{-x})/(e^x + e^{-x})，输出 (-1,1)，零中心。缺点：仍存在梯度消失。

### ReLU

f(x) = max(0, x)。优点：计算简单、缓解梯度消失（正半区导数为 1）、稀疏激活。缺点：负半区梯度为零，神经元可能永久"死亡"（Dead ReLU）。

### GELU（Gaussian Error Linear Unit）

GELU(x) = x · Φ(x)，其中 Φ 是标准正态分布的累积分布函数。是 BERT、GPT 等模型的标准激活函数。相比 ReLU 更加平滑，在 x=0 附近有非零导数，提供了概率性的门控机制。

### Swish / SiLU

Swish(x) = x · σ(βx)。β=1 时即 SiLU（Llama 系列使用的激活函数）。平滑、非单调，在实践中常优于 ReLU。

### GLU（Gated Linear Unit）

GLU 不是一种激活函数，而是一种**神经网络层结构**——线性变换后接门控机制：

```
GLU(X) = (XW + b) ⊙ σ(XV + c)
```

将输入经两个线性变换后，一部分经 sigmoid 来决定每个维度"信息能通过多少"，另一部分作为内容。这种门控机制让模型学会选择性传递信息。

### SwiGLU

将 GLU 中的 sigmoid 替换为 Swish/SiLU：

```
SwiGLU(X) = (XW + b) ⊙ SiLU(XV + c)
```

SwiGLU 是 Llama 系列、PaLM 等主流大模型使用的 FFN 激活方式，综合了 GLU 的门控能力和 Swish 的平滑性。

## 正则化：L1 与 L2

**L1 正则化**：λ Σ |w_i|（权重绝对值之和）。产生稀疏权重解，适合特征选择——使不重要的特征权重趋近于 0。

**L2 正则化**：λ Σ w_i²（权重平方之和）。使大权重减小，产生平滑解，防止过拟合。在 SGD 中，Weight Decay 等价于 L2 正则化。

## 常见损失函数

### 均方误差（MSE）

适用于回归任务，预测连续数值。MSE 形式上和方差一样，但是用来计算损失——衡量预测值与真实值之间的平方距离。

### 交叉熵损失（Cross-Entropy）

**为什么 SFT 的 loss 是 CE 而不是 MSE？**

SFT 预测词表，本质是一个**分类任务**，而不是连续值回归。
- **交叉熵（CE）**：用于概率分布之间的匹配（分类问题），梯度与 softmax 输出和标签之差成正比，优化高效
- **MSE**：用于连续数值之间的距离（回归问题）

token 是离散符号（one-hot），天然适配 CE。真实 one-hot 分布下交叉熵计算等价于最大化正确 token 的对数概率（MLE）——最小化交叉熵等于最大化 log likelihood。

**如果用 MSE 会有什么问题？**
1. **不符合概率建模目标**：MSE 在做"让概率向量接近 one-hot 向量"，但我们真正想要的是"最大化正确 token 的对数似然"，两者目标函数不一致
2. **梯度问题**：CE 的梯度是 log(softmax) - y，是线性的；MSE 的梯度包含 sigmoid-like 的导数，当预测远离目标时 CE 梯度更大，学习更快
3. **高维稀疏空间不友好**：词表是 50k+ 维，one-hot 只有一个 1 其余全 0。MSE 会对所有维度平均惩罚，而 CE 只关注正确类别——CE 更"稀疏友好"，计算和优化更高效

### Focal Loss

**公式**：FL(p_t) = -α_t · (1 - p_t)^γ · log(p_t)

- α：平衡正负样本的权重因子
- γ：聚焦参数，调节难样本的权重。γ=0 退化为标准 CE；γ 越大，对已分类正确样本的降权越强
- **设计目的**：降低易分类样本的权重，让模型更关注难分类样本。解决 one-stage 检测中正负样本极度不平衡问题（如 1:1000）

### Pairwise Loss（Reward Model）

```python
class PairwiseLoss(nn.Module):
    def forward(self, chosen_reward, reject_reward, margin=0.0):
        loss = -F.logsigmoid(chosen_reward - reject_reward - margin)
        return loss.mean()
```

**特性**：PairwiseLoss 有个良好的属性——**自带 margin 效果**。因为在 0 附近也会产生一定的 loss，来更新模型，拉大正负例的差距。

**-logsigmoid 曲线理解**：当 `chosen_reward - reject_reward < 0` 时 loss 急剧上升，表示正例样本得分小于负例样本得分，产生较大 loss，回传产生较大梯度更新模型。当 `chosen_reward - reject_reward > 0` 时 loss 基本趋近于 0，表示正例得分大于负例得分是合理的，不产生 loss。

### SFT Loss（GPT Language Model Loss）

核心：`logits[:, :-1, :]` 预测 `labels[:, 1:]`。自回归语言模型用当前 token 预测下一个 token。输入 `[w1, w2, w3, w4]`，预测 `w1→w2, w2→w3, w3→w4`。SFT 时通常把 prompt 部分的 loss mask 掉，只学习回答部分。

## 信息论基础

### 自信息（Self-Information）

**定义**：
```
I(x) = log(1/P(x)) = -log P(x)
```

衡量单个结果携带的信息量或意外程度。`(1/P(x))` 代表发生概率为 P(x) 的事件发生时的**惊讶程度**。与概率呈负相关——概率越大，发生时越不惊讶，信息量越小；概率越小，发生时越惊讶，信息量越大。加上 log 是因为 log 具有可加性，方便概率连乘的计算。

**直观理解**：如果某事件几乎必然发生（P≈1），那么观察到它发生时我们几乎不惊讶，信息量趋近于 0；如果某事件几乎不可能发生（P≈0），观察到它时我们会非常惊讶，信息量很大。

---

### 熵（Entropy）

**定义**：
```
H(P) = -Σ_x P(x) log P(x) = E_{x~P} [-log P(x)]
```

熵是信息论中衡量随机变量不确定性的指标，表示一个概率分布的混乱程度。**熵越高，不确定性越大。** 衡量分布的平均意外程度——计算以概率为权重的意外程度的期望值。

**直观理解**：熵代表使用最佳编码来编码结果所需的平均比特数，也是"平均意外程度"。
- 熵越高：概率分布越均匀，平均编码所需比特数越高，不确定性越大
- 熵越低：概率分布越集中，平均编码所需比特数越低，不确定性越小

**性质**：
- 非负性：H(P) ≥ 0，当且仅当 P 为确定性分布（one-hot）时取等
- 均匀分布时熵最大
- 可加性：独立变量的联合熵等于各变量熵之和

---

### 交叉熵（Cross-Entropy）

**定义**：
```
H(P, Q) = -Σ_x P(x) log Q(x)
```

衡量两个概率分布 P（真实分布）和 Q（预测分布）之间的差异。如果是用错误的概率分布 Q 来对真实事件进行编码，需要的编码数量更多，因此使用的平均比特数更大。

**核心关系**：
```
H(P, Q) = H(P) + D_KL(P || Q)
```

交叉熵 = 熵 + KL 散度。

**直观理解**：
- 用 Q 来编码 P 分布的样本，平均需要多少比特
- Q 越接近 P，交叉熵越小
- 当 Q = P 时，交叉熵等于熵 H(P)——这是理论最小值
- 交叉熵衡量预测分布距离真实分布的偏离大小：用 Q 来描述真实分布 P 所需要的"平均总成本"或"非对称距离"（区别于欧几里得距离——两点之间直线最短）

**与 one-hot 标签的关系**：当真实标签是 one-hot 分布时，NLL Loss = Cross-Entropy Loss。负对数似然损失是让惊讶值变小，所以 NLL 其实是 CE 的一个特例。当 P 是 one-hot 编码时，交叉熵退化为仅优化模型对正确类别的预测概率。

**在模型优化中，最小化交叉熵损失等价于最小化预测分布与真实分布的 KL 散度**（因为真实分布的熵 H(P) 是常数）。

---

### KL 散度（Kullback-Leibler Divergence）

**定义**：
```
D_KL(P || Q) = Σ_x P(x) log(P(x) / Q(x))
             = H(P, Q) - H(P)
```

**核心含义**：KL 散度衡量从分布 Q 到分布 P 的**信息损失**。H(P) 表示自身的原始的信息量，减去 H(P,Q) 等于原本的基础上损失的信息量。KL 散度可以被定义为两个概率分布之间的交叉熵与真实分布的熵的差值。

**形象理解**：KL 散度 = 额外浪费的比特数——衡量实际数据遵循真实分布 P 时，使用错误分布 Q 来编码数据所产生的额外成本。也可以理解为"认知偏差"——交叉熵减去自信息量，交出的额外成本。

**关键特性**：

1. **非负性**：D_KL(P || Q) ≥ 0，等号当且仅当 P(x) == Q(x) 时成立（吉布斯不等式）

2. **可加性**：对于独立分布，D_KL(P1·P2 || Q1·Q2) = D_KL(P1 || Q1) + D_KL(P2 || Q2)

3. **不对称性**：D_KL(P || Q) ≠ D_KL(Q || P)。**不能把 KL 散度当作"距离函数"使用**——换方向会得到不同结果，换方向后的逼近的"代价"是不同的。

4. **概率分布敏感**：由于 P(x) 的概率加权，导致高概率区域的差异被放大，低概率区域的差异被抑制。P(x)/Q(x) 代表点 x 处两个分布的偏离程度。

---

### 正向 KL vs 反向 KL

**正向 KL**：D_KL(P || Q) = Σ P(x) log(P(x)/Q(x))

**反向 KL**：D_KL(Q || P) = Σ Q(x) log(Q(x)/P(x))

**Motivation——为什么同样是真实分布和预测分布矛盾的情况，P 大 Q 小时对交叉熵的数值贡献大，而 P 小 Q 大时数值却这么小？**

因为交叉熵 H(P,Q) = -Σ P log Q，由真实分布 P 加权：
- P 大 Q 小 → P(x) × (-log Q(x)) 很大，因为 -log Q(x) 很大且被 P(x) 放大
- P 小 Q 大 → P(x) × (-log Q(x)) 很小，因为被 P(x) 抑制

这引出了正向 KL 和反向 KL 的本质差别：

| 形式 | 公式 | 关注点 | 模型性格 | 适用场景 |
|------|------|-------|---------|---------|
| 正向 KL | Σ P log(P/Q) | "别漏掉正确答案" | **博学、包容**（覆盖所有可能） | SFT、预训练、知识蒸馏 |
| 反向 KL | Σ Q log(Q/P) | "别乱说话，说准就行" | **保守、专一**（锁定单个模式） | RLHF 惩罚项、生成多样性控制 |

**正向 KL（D_KL(P || Q)，P=真实，Q=模型）**：
- 要求预测的 token 中，真实的 token 的概率一定要大，否则给很大惩罚
- 让模型尽可能不漏真实高概率 token，可能会给尽可能多的地方都给高概率
- 行为：倾向于"覆盖所有模式"，即使导致概率分散
- **鼓励 Q 往 P 大的地方增加，即使漏了大的 P 没关系，至少 Q 不应该在小的 P 上增加（重罚）**
- 要求模型能够模仿真实的模式，锁定模式

**反向 KL（D_KL(Q || P)，Q=模型，P=真实）**：
- 要求模型变得保守，绝对不能说出真实分布小的 token，否则惩罚很大
- 因此模型变得保守，只要不乱说，KL 散度就会变得小
- 行为：倾向于"锁定单个模式"，Q 在 P 的一个峰值附近，即使漏掉其他模式
- **生成多样性控制，防止熵坍塌，保持多样性**

**为什么 GRPO/PPO 使用的是反向 KL？**

SFT 模型通常是范围分布很广的（分布较均匀），而 Reward Model 会让分布变得尖锐（Q 峰 >> P 峰）。使用反向 KL 约束是从尖锐分布变为多样分布，即让 policy model 追随参考模型中大概率的动作，而不在意小概率的动作。此时 policy model 是 P，ref model 是 Q。通过这样的方式，允许小概率进行探索，而禁止原始的大概率操作变小——**不一定所有模式都覆盖**。

---

### 总结：交叉熵与 KL 散度在应用上的区别

**交叉熵损失的应用**：主要用于监督学习中的分类任务（如 SFT），目标是直接最小化模型预测与标签之间的差异。由于标签通常是确定性的（one-hot），交叉熵直接优化模型对正确类别的置信度。当真实分布 P 为确定性分布（如独热编码）时，H(P)=0，此时交叉熵损失等价于 KL 散度。

**KL 散度的应用**：
- **模型蒸馏**：让学生模型的输出分布 Q 匹配教师模型的分布 P，使用 KL(P || Q)
- **强化学习（如 PPO/GRPO）**：约束新旧策略的差异（KL(P_old || P_new)），防止策略突变
- **生成模型**：衡量生成分布与真实分布的距离（如变分推断中 KL(Q || P)）

---

### k1 / k2 / k3 —— KL 散度的蒙特卡洛估计

**为什么需要近似估计？**

因为离散的概率分布在大模型词表中甚至有 200k 个 token，计算量巨大。以及对于连续的概率分布，除了高斯分布以外，其他无法求得显式解。因此采用**蒙特卡洛近似值**来估计。

我们对估计的 KL 散度有两个要求：
1. 第一最好是无偏的，即估计值的期望与真实值相等
2. 第二方差要尽可能小

> **方差与偏差的直观解释**：
> - **偏差**：重复很多次估计（比如采样多组数据），每次的结果可能不一样，但这些结果的**平均值是对的**
> - **方差**：方差小意味着"每次估计都很稳定，基本落在真实值附近"

设 `r = π_ref(x) / π_θ(x)`（从当前策略采样的样本在参考模型与当前模型的概率比），`log r = log π_ref(x) - log π_θ(x)`。

---

#### k1 估计

**公式**：`k1 = -log r`

k1 的形式就是按定义来的，所以它显然是无偏的。但是因为 log 函数不是恒正的，由于有负值，因此方差大，k1 不满足低方差要求。

**特性**：**低偏差，高方差**

---

#### k2 估计（f-散度近似）

**公式**：`k2 = (log r)² / 2`

k2 基于 f-散度，因为是恒正的（平方），所以方差相比 k1 会更小。虽然这会带来一定的偏差，但是偏差很小。k2 和 k1 的二阶导 f''(1) 都是 1，所以可以近似估计。

**缺陷**：当 p 和 q 的偏差与 0 相差比较大时，采样的数量再多，获得的 KL 散度并不收敛于真实值。对分布差距比较大的两个分布来说才是高偏差。

**特性**：**高偏差，低方差**

---

#### k3 估计（Control Variate 方法）

**公式**：`k3 = (r - 1) - log r`

更进一步，我们能不能既要无偏，又要低方差呢？想要降低方差，通常的办法是**控制变量（control variate）法**，即选用无偏的 k1，但是需要再加上一些期望为 0、且与 k1 负相关的项，从而在保证无偏的同时降低方差。(r-1) 就是一个期望为 0 的项。

所以对于任意 λ，`-log r + λ(r-1)` 都是一个无偏估计（r-1 的期望为 0）。因此接下来选一个 λ 使这个公式方差最小。因为 `log(x) ≤ x - 1`，所以能够保证该式恒正，能够尽量减小方差。x-1 刚好是 log(x) 在 x==1 处的切线。

**特性**：**无偏、低方差**——兼顾了 k1 和 k2 的优点。这也是 GRPO 等大模型 RL 训练中的常用选择。

---

#### 三种估计量对比

| 估计量 | 公式 | 偏差 | 方差 | 说明 |
|-------|------|-----|------|------|
| k1 | -log r | 无偏 | 高 | 按定义直接估计，log 可取负值导致方差大 |
| k2 | (log r)² / 2 | 有偏 | 低 | f-散度二阶近似，恒正 |
| k3 | (r - 1) - log r | 无偏 | 低 | Control variate，实际常用 |

**验证代码**：
```python
import torch
import torch.distributions as dis

p = dis.Normal(0, 1)
q = dis.Normal(0.1, 1)
x = q.sample((10_000_000,))
true_kl = dis.kl_divergence(q, p)
logr = p.log_prob(x) - q.log_prob(x)

k1 = -logr
k2 = logr ** 2 / 2
k3 = (logr.exp() - 1) - logr

for name, k in [("k1", k1), ("k2", k2), ("k3", k3)]:
    bias = (k.mean() - true_kl) / true_kl
    std  = k.std() / true_kl
    print(f"{name}: bias={bias:.4f}, std={std:.4f}")
```

> (标准差/方差) 越大，代表分布离平均值越远，曲线整体看起来更平坦
> (标准差/方差) 越小，代表分布越靠近均值，即均值附近的概率更大，曲线整体看起来更尖锐

### 最大似然估计（MLE）与负对数似然

**最大似然估计**：给定数据，寻找使数据出现概率最大的参数。

```
L(θ) = Π_i P(y_i | x_i; θ)
```

xi 是输入，yi 是 ground truth。这里计算的是分类问题中每个输入对应 ground truth 的概率的连乘（即每个样本都计算正确的概率的连乘）。最后结果越大越好。

**负对数似然损失**：最大似然估计等价于最小化负对数似然（取负号将最大化变为最小化，取对数将连乘变为连加）：

```
NLL = -Σ_i log P(y_i | x_i; θ)
```

y 是 ground truth，ŷ 是预测值。负对数似然损失通常被称为**交叉熵损失（cross-entropy loss）**。由于 y 是一个长度为 q 的独热编码向量，所以除了一个项以外的所有项 j 都消失了。由于所有 ŷ_j 都是预测的概率，所以它们的对数永远不会大于 0。因此，如果正确地预测实际标签，即如果实际标签 P(y|x)=1，则损失函数不能进一步最小化。

在拟合一个模型到真实分布上时，使用 KL 散度。因为前一项真实分布的熵是一个常数，因此优化 KL 散度其实也就是优化交叉熵这一项。另一个角度来说，拟合一个真实分布，也相当于最小化模型对真实分布的惊讶程度（NLL）。

## Bert

BERT 同时进行两个预训练任务：
1. **MLM（Masked Language Modeling，掩码语言模型）**：随机遮盖 15% 的 token，让模型预测被遮盖的词（完型填空）
2. **NSP（Next Sentence Prediction，下一句预测）**：判断句子 B 是否是句子 A 的下一句（判断蕴含关系）

### RoBERTa 相比 BERT 的改进

1. **训练数据与规模扩大**：数据量增加至约 160GB，远超 BERT 的原始数据量，从而提升模型对多样化文本的适应能力；通过更长的训练周期、更大的批量大小（batch size）和动态调整的学习率，使模型收敛更充分

2. **训练策略改进**：
   - **动态掩码（Dynamic Masking）**：BERT 在预处理时对文本进行静态掩码（固定掩码位置），而 RoBERTa 在每次输入时动态生成掩码，增强模型对不同掩码模式的鲁棒性
   - **移除 NSP 任务**：RoBERTa 取消了 BERT 中的下一句预测任务，专注于优化掩码语言模型（MLM），简化训练目标

3. **模型细节优化**：改进优化器的超参数（如学习率调度策略），并采用更高效的训练技巧（如**混合精度训练**），提升训练效率和模型性能

4. **性能表现**：通过上述改进，RoBERTa 在多项下游任务（如 GLUE 基准）中超越 BERT，验证了其优化策略的有效性

**一句话总结**：RoBERTa 通过"**数据量扩展 + 动态掩码 + 训练策略简化**"三大核心改进，在保持 BERT 架构的基础上显著提升性能，体现了预训练模型对数据和训练细节的敏感性。

## PEFT

### LoRA（Low-Rank Adaptation）及其变体

**LoRA 核心思想**：在预训练权重旁添加低秩分解矩阵（A × B），只训练这些增量矩阵，冻结原始权重。利用"模型微调中的权重更新是低秩的"这一假设。

**公式**：`h = W_0 x + ΔW x = W_0 x + B A x`，其中 B ∈ R^{d×r}，A ∈ R^{r×d}，r 是秩（通常很小，如 8 或 16）。

**初始化方式**：A 用高斯分布初始化，B 用零初始化。这样保证训练初始时 ΔW = 0，不破坏预训练权重。

**LoRA 变体**：
1. **AdaLoRA**：自适应地为不同层分配不同的秩，更高效地利用参数预算
2. **LoRA+**：对矩阵 A 和 B 设置不同的学习率，进一步提升收敛效果
3. **rsLoRA**：重新缩放的初始化方式，改进训练稳定性
4. **QLoRA**：4-bit NormalFloat 量化 + 双重量化 + 分页优化器，在单个 GPU 上微调 65B 模型成为可能

### Soft Prompt Tuning 总结

| 方法 | 插入位置 | 可训练参数 | 适用场景 |
|------|---------|-----------|---------|
| Prompt Tuning | 仅输入 embedding 前 | 极少 | 简单任务 |
| Prefix Tuning | 每层 KV cache 前 | 中等 | 各类生成任务 |
| P-Tuning | 输入层 + prompt encoder | 中等 | 知识密集型任务 |
| P-Tuning v2 | 每层 hidden state 前 | 较多 | 复杂任务，接近全参微调 |

**核心区别**：
- **Prompt Tuning**：只在 embedding 前面加入 virtual token，只训练这些 virtual token 的 embedding
- **Prefix Tuning**：在每层的 KV cache 前面加上前缀，影响注意力
- **P-Tuning**：virtual token 需要随机初始化，经过 prompt encoder（如 LSTM）进行编码，得到优化后的具有强语义的 virtual token。这里只更新原始随机初始化的 virtual token 以及 prompt encoder
- **P-Tuning v2**：类似 Prefix Tuning，每层都插入参数，但是这里是每一层插入 virtual token 在 hidden state 的前缀

## MoE 模型架构

### Mixture of Experts 核心思想

将标准 FFN 层拆分为多个并行的"专家"子网络。每个 token 通过门控网络（Router/Gate）仅路由到 top-k 个专家进行计算，其余专家被跳过。

**优点**：
- **计算量与参数量解耦**：总参数量极大（所有专家之和），但每个 token 的计算量仅取决于激活的专家数
- **条件计算**：不同 token 激活不同专家，隐式实现任务专门化

### Mixtral 8×7B

- 8 个专家，每个 token 选择 top-2 个专家
- 每个专家为标准 FFN 层
- 门控网络：一个 Linear 层输出 8 个 logits，经 softmax 后选 top-2
- 质量相当于 Llama 2 70B，推理速度快约 6 倍
- 总参数量约 47B，但每个 token 仅激活约 12B 参数

### DeepSeek MoE

**创新点**：
1. **共享专家（Shared Experts）**：部分专家被所有 token 强制激活，承担通用知识，减轻路由专家的负担
2. **辅助损失（Load Balancing Loss）**：添加 load balancing loss，确保专家被均衡使用，防止"专家坍塌"
3. **多尺度/细粒度专家**：相比传统 MoE 使用更多但更小的专家，提升专业化程度

### 关键技术挑战

**1. 负载均衡**：问题——某些专家被过度使用（路由热点），某些专家被闲置。解决方案——添加 auxiliary load balancing loss：`L_balance = α · Σ(f_i · g_i)`，其中 f_i 为专家 i 实际被选中的频率，g_i 为门控分配给专家 i 的平均概率。

**2. 路由机制**：
- **硬路由（Top-k）**：选 k 个专家，离散不可导，需配合梯度估计（straight-through estimator）
- **软路由（加权和）**：所有专家参与，连续可导，但计算量巨大

**3. 训练稳定性**：专家坍塌（expert collapse）——所有 token 都路由到少数专家。对策：合理初始化门控权重、添加负载均衡损失、注入路由噪声。

## 量化与混合精度

### FP16 vs BF16

| 格式 | 精度位（尾数） | 指数位 | 特点 |
|------|-------------|-------|------|
| FP16 | 10 | 5 | 精度较高，动态范围小（约 6.5×10⁴），易上溢/下溢 |
| BF16 | 7 | 8 | 指数位与 FP32 相同（8 位），动态范围大（约 3.4×10³⁸） |

- FP16 的指数位比较少，因此不能表示很小的数和很大的数，容易出现数字上溢和下溢的问题。在半精度训练或推理时，如果使用 FP16 需要使用 loss scaling 来缓解
- BF16 的指数位和 FP32 相同，因此相当于砍掉精度的 FP32。不需要 loss scaling
- 一般情况下使用 BF16 是更稳定的

### 混合精度训练

**核心思想**：维护一份 FP32 的主权重。在前向和反向传播时都使用半精度（BF16/FP16）来计算（加速计算、节省显存），更新时合并到 FP32 主权重中保持精度不变。

**优势**：
- 节省显存（约 2x）
- 加速训练和推理，计算量更低

**Loss Scaling（损失缩放）**：FP16 的动态范围有限（最小正数约 6×10⁻⁸），小梯度容易下溢为 0。解决方法：在前向时将 loss 乘以一个大缩放因子（如 128），反向传播后再将梯度缩小回来。BF16 不需要此操作。

### 各精度格式对比

| 精度格式 | 比特数 | 相对 FP32 显存 | 适用场景 |
|---------|-------|-------------|---------|
| FP32 | 32 | 1× | 优化器状态、梯度累加 |
| TF32 | 19 | 1.7× | 矩阵乘（NVIDIA Ampere+） |
| BF16 | 16 | 2× | 前向/反向传播（主流默认） |
| FP16 | 16 | 2× | 需要较高精度时 |
| FP8 | 8 | 4× | 推理加速（需校准） |
| INT8 | 8 | 4× | 量化推理 |
| INT4 | 4 | 8× | 极端显存受限 |

## 分布式训练

### DeepSpeed ZeRO（Zero Redundancy Optimizer）

**ZeRO Stage 1 —— 优化器状态分片**：将 Adam 的 m（一阶动量）和 v（二阶动量）分片到各 GPU。显存节省约 4×，通信开销很小。

**ZeRO Stage 2 —— 梯度分片**：Stage 1 + 梯度也进行分片。显存节省约 8×，通信开销中等。

**ZeRO Stage 3 —— 参数分片**：Stage 2 + 模型参数本身也进行分片。显存节省接近线性于 GPU 数量，通信开销较大。

**Offloading**：
- **ZeRO-Offload**：将优化器状态和梯度卸载到 CPU 内存中
- **ZeRO-Infinity**：更进一步，利用 NVMe SSD 作为扩展存储

**显存构成图示理解**（全参微调，每参数）：
```
蓝色块：模型参数（Parameters）
橙色块：梯度（Gradients），即模型反向传播后计算出的参数的更新方向
绿色块：优化器状态（Optimizer States），比如 Adam 优化器中的一阶动量和二阶动量

FP32 参数(4B) + BF16 参数(2B) + 梯度(2B) + m(4B) + v(4B) = 16 字节/参数
```

### Megatron-LM：张量并行（TP）

将单层 Transformer 内的权重矩阵沿 hidden 维度切分到多个 GPU。例如 FFN 层 `[d_model, 4*d_model]` 切分为 N 份，每 GPU 计算 `4*d_model/N` 的部分。QKV 投影同样切分。

### 其他并行策略

| 并行策略 | 原理 | 瓶颈 |
|---------|------|------|
| DP（数据并行） | 每 GPU 持有完整模型副本，数据分片 | 梯度 all-reduce 同步通信量大 |
| PP（流水线并行） | 不同层放不同 GPU，micro-batch 流水线执行 | 流水线气泡（bubble），GPU 利用率不高 |
| EP（专家并行） | 每个专家放在不同 GPU，token 路由到对应专家 GPU | all-to-all 通信 |
| TP（张量并行） | 层内矩阵沿 hidden 维度切分 | NVLink 限制在同一节点内 |

### 3D 并行 = DP + PP + TP

大模型训练的标准配置：
- TP（节点内，NVLink）：8 GPU，张量并行
- PP（节点间）：减少单节点显存压力
- DP（跨节点）：扩展到更大规模

## 采样策略

### 温度（Temperature）

在 softmax 前将 logits 除以温度 T：`p_i = exp(z_i/T) / Σ exp(z_j/T)`

- **T → 0**：趋近 argmax（贪婪解码），确定性最强
- **T = 1**：原始概率分布
- **T → ∞**：趋近均匀分布，随机性最强

### 与 Top-K / Top-P 结合

- **低温度 + Top-K=5**：在少量候选词中选择最可能的结果，适合高确定性任务
- **高温度 + Top-P=0.9**：在高概率词集中引入多样性，适合创意生成

## 传统机器学习基础

### 随机森林

集成多棵决策树，通过 Bootstrap 采样（随机选样本）和随机子空间（随机选特征）增加树之间的差异性，最终投票（分类）或平均（回归）。**优点**：准确率高、能处理高维数据、可并行训练快、对异常值不敏感、不易过拟合。**缺点**：模型体积大、解释性不如单棵决策树、对小样本可能不如 SVM。

### 决策树

递归地根据特征值分裂数据集，构建树形结构进行决策。

**分裂准则**：
- **ID3**：信息增益 Gain(D, A) = H(D) - Σ(|D_v|/|D|) H(D_v)（偏向多值特征）
- **C4.5**：信息增益率 GainRatio(D, A) = Gain(D, A) / IV(A)（解决 ID3 偏向问题）
- **CART**：Gini 系数 Gini(D) = 1 - Σ p_k²（分类）或 MSE（回归）

**剪枝策略**：预剪枝（分裂过程中就停止，如样本数太少、增益太小）和后剪枝（先构建完整树再自底向上剪去不必要的分支）。

### 朴素贝叶斯

基于贝叶斯定理 P(Y|X) = P(X|Y) P(Y) / P(X)，加上**特征条件独立假设** P(X|Y=k) = Π P(x_i|Y=k)。**优点**：计算简单、速度快、对缺失数据不敏感、适合高维数据。**缺点**：特征独立假设在现实中往往不成立，可能影响性能。**变种**：高斯朴素贝叶斯（连续特征）、多项式朴素贝叶斯（计数特征，如文本分类）、伯努利朴素贝叶斯（二值特征）。

### SVM（支持向量机）

**核心思想**：找到一个超平面，最大化不同类别之间的间隔（margin）。

**硬间隔 SVM**：min ½||w||² s.t. y_i(w·x_i + b) ≥ 1, ∀i

**软间隔 SVM**（允许部分样本错分，引入松弛变量）：min ½||w||² + C Σ ξ_i s.t. y_i(w·x_i + b) ≥ 1 - ξ_i, ξ_i ≥ 0

**核技巧**：将低维数据映射到高维空间，使其线性可分，但不直接计算高维坐标，而是通过核函数计算内积。常用核函数：线性核 K(x,y) = x·y、多项式核 K(x,y) = (x·y+1)^d、RBF 核 K(x,y) = exp(-γ||x-y||²)、Sigmoid 核 K(x,y) = tanh(αx·y+c)。

**优缺点**：优点是在高维空间有效、泛化能力强、有严格的数学理论；缺点是训练慢 O(N³)、不适合大规模数据、对参数和核函数敏感。

## 优化器

### SGD（Stochastic Gradient Descent）

**原理**：每次迭代只使用一个样本或小批量数据来估计损失函数的梯度并更新参数。计算成本低、更新频率高，具有一定的随机性能够帮助模型跳出局部最优。

**SGD with Momentum**：模拟物理中的惯性，引入历史梯度的累积项。`v_t = γ v_{t-1} + η ∇L(θ_t)`，θ_{t+1} = θ_t - v_t。γ 通常取 0.9。**直观理解**：像小球滚下山坡，动量让它在平坦区域保持速度，在梯度方向持续加速，在震荡方向保持平滑。

**Weight Decay 与 L2 正则**：在 SGD 中加入 Weight Decay 实际等价于对参数引入 L2 正则项，即在目标函数中加入 λ|θ|²，从而惩罚模型参数过大，防止过拟合。

**SGD vs Adam**：

| 方面 | SGD | Adam |
|------|-----|------|
| 学习率 | 固定或调度 | 自适应（每个参数独立） |
| 速度 | 慢 | 快（尤其在 early stage） |
| 稳定性 | 依赖动量，震荡大 | 更稳定 |
| 内存消耗 | 少 | 多（需存梯度一阶与二阶动量） |
| 泛化能力 | **通常更好**（尤其在 CV 中） | 易过拟合 |
| 超参数 | 少（lr, momentum） | 多（lr, β1, β2, ε） |

**口诀**：`图像用 SGD，语言用 Adam；收敛快 Adam 稳，泛化强 SGD 顶`

### Adam

像 SGD 只利用当前的梯度对模型进行更新，容易使模型的参数发生剧烈变化，导致训练困难，不容易收敛。

**Adam = Momentum + Adaptive Learning Rate**

```
m_t = β₁ m_{t-1} + (1-β₁) g_t       # 一阶动量（梯度指数滑动平均）→ 总体下降方向
v_t = β₂ v_{t-1} + (1-β₂) g_t²      # 二阶动量（梯度平方指数滑动平均）→ 梯度变化程度
m̂_t = m_t / (1-β₁^t)                 # 偏差修正
v̂_t = v_t / (1-β₂^t)
θ_t = θ_{t-1} - η · m̂_t / (√v̂_t + ε)
```

一般 β₁=0.9，β₂=0.999。

| 成分 | 数学名称 | 直观意义 | 作用 |
|------|---------|---------|------|
| m_t | 一阶动量（梯度平均） | 总体下降方向 | 减少震荡，快速收敛 |
| v_t | 二阶动量（方差估计） | 梯度变化程度 | 自适应调整步长 |
| 偏差修正 | Bias Correction | 矫正初期偏差 | 使初期估计更准、梯度不为零 |

**一句话**：方向靠 m_t，步长靠 v_t，启动靠修正。

vt 代表梯度的方差——方差小，学习率相当于变大，说明往这个方向更新的确定性是比较大的，所以可以更新比较大。

**Adam 的问题**：自适应学习率可能导致后期收敛不稳定、权重衰减方式可能不对（见 AdamW）、泛化性能可能比 SGD 差。

### AdamW

| 维度 | Adam | AdamW |
|------|------|-------|
| 权重衰减方式 | 加进梯度中 | 分离出来，直接对参数衰减 |
| 是否扰动动量 | ✅ 是 | ❌ 否 |
| 是否等价 L2 正则 | ❌ 不等价 | ✅ 完全等价 |
| 大模型适用性 | 容易不稳定 | BERT/GPT 默认优化器 |

**本质区别**：原始 Adam 把权重衰减加在梯度中，这种方式会污染动量估计，也不符合 L2 正则的理论。AdamW 通过把正则项从梯度中分离出来，单独对参数做衰减，使优化更稳定，也更适合大模型训练。

```python
# 错误：Adam 中的权重衰减（加在梯度上）
grad = grad + wd * w

# 正确：AdamW 解耦权重衰减（直接加在参数上）
w = w - lr * (grad + wd * w)  # 衰减不影响 grad 和动量
```

## 指标 Metric

### 困惑度（Perplexity, PPL）

困惑度是一种用来评价语言模型好坏的指标。直观上理解，当我们给定一段**非常标准的、高质量的、符合人类自然语言习惯的文档作为测试集**时，模型生成这段文本的概率越高，就认为模型的困惑度越小，模型也就越好。

**定义**：
```
PPL = exp(-(1/N) Σ_{i=1}^N log P(w_i | context))
```

最后的值越小，模型越好。

**直观理解**：困惑度可以理解为模型在预测下一个 token 时的"平均分支因子"——即模型平均在每个位置需要在多少个等概率选项中做选择。例如 PPL=10 意味着模型平均在 10 个等可能的选项中做选择。

**与交叉熵的关系**：困惑度的计算其实依赖于语言模型输出 token 的概率分布，因此在比较语言模型时，其实是在比较哪个分布更符合测试集的真实概率分布。对测试集困惑度越小的模型，交叉熵也会越小——都是针对 ground truth 和预测分布进行计算的。

**计算注意事项**：由于所有模型都只能接受有限长的序列，因此可以采用等长切分或者滑动窗口的方式决定上下文。

## 模型显存估算

### 静态部分
1. 模型参数：FP32 主副本（4 字节）+ BF16 副本（2 字节）
2. 优化器参数：Adam 的一阶动量 m（FP32，4 字节）+ 二阶方差 v（FP32，4 字节）
3. 梯度：BF16（2 字节）

### 动态部分
中间激活值，动态变化。近似公式：`layer_num × 2 × batch_size × seq_len × num_head × head_dim × 2B`

### LoRA 为什么相比全参微调节省显存

预训练权重的优化器状态和梯度都不需要计算，因此大大降低了显存的占用。冻结的预训练权重无需存储对应的优化器状态和梯度，仅需为少量 LoRA 参数（A 和 B 矩阵）维护这些开销。

### 快速估算（7B 模型全参微调）

| 组成部分 | 计算 | 显存 |
|---------|------|------|
| 模型参数（FP32 + BF16） | 7B × (4 + 2) B | 42 GB |
| 优化器状态（m + v） | 7B × (4 + 4) B | 56 GB |
| 梯度（BF16） | 7B × 2 B | 14 GB |
| 激活值（动态） | 取决于 batch_size、seq_len | 数 GB ~ 数十 GB |
| **合计** | — | **约 112 GB+** |

LoRA 微调（r=8）：仅需约 14GB（冻结参数的 BF16 副本）+ 少量可训练参数的优化器开销。
