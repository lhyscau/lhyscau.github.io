## 强化学习基础

### 基本概念

**原始的策略梯度**：强化学习的目标是最大化期望累积回报。

某个 trajectory（轨迹）在 policy model 的期望为 p，奖励为 R。trajectory 其实是每一步针对当前状态，对下一个状态的预测得到的：

```
J(θ) = E_{τ~π_θ} [R(τ)]
∇J(θ) = E_{τ~π_θ} [ Σ_{t=0}^T ∇ log π_θ(a_t | s_t) · R(τ) ]
```

**核心直觉**：根据当前 trajectory 的 return 值是否是大于 0 的——如果大于 0，则针对（优化）policy model，增大 trajectory 中每个 state 下动作的概率；否则减小。以此来提高 return 回报的数学期望。

> tips：这里把导数符号拿走，是为了分析单调性。导数的不同原函数，单调性都是一样的。

### 策略梯度定理的两个改进

**Q1**：在状态 state 下做动作 action 的 reward 应该由该 step 到最后 step **累积**的 reward 来决定，而不是由一整个 trajectory 累积的 reward 来决定。因为一个动作只能影响后续的 reward，而不能影响前面的 reward。

**Q2**：针对 Q1 做出改进，当前步只会影响后几步，并且影响会慢慢衰减，而不是对当前步到最后一步的影响都一样大。

**针对 Q1 & Q2 的解决方案**：使用缩放因子 γ（gamma）来缩放，当前步离 t 越远，受到 t step 做出 action 的影响越小。尽可能表现**当前动作**对 trajectory return 的影响，避免其他因素：

```
R_t = Σ_{k=0}^{∞} γ^k r_{t+k+1}
∇J(θ) = E_{τ~π_θ} [ Σ_{t=0}^T ∇ log π_θ(a_t | s_t) · R_t ]
```

原本的概念是一整个 trajectory 的概率 × 该 trajectory 得到的回报，变成了 trajectory 每一个 step 概率 × 该 step 的 return 的乘积和。

### Actor-Critic 框架

**问题**：在好的局势下，所有动作的概率都会增加；在坏的局势下，所有动作的概率都会减小。这样会导致训练速度变慢。

**解决方案**：最好是让相对好的动作概率增加，相对差的动作概率减小，这样加快训练速度。为此，对所有动作的概率都减去一个 baseline：

```
∇J(θ) = E_{τ~π_θ} [ Σ_{t=0}^T ∇ log π_θ(a_t | s_t) · (R_t - b(s_t)) ]
```

这样在不同的局势下，采取不同的动作得到的 reward 有正也有负。在 PPO 中，这里的 Baseline 就是 Critic Model 生成当前状态的价值 V(s_t)。

- **Actor（策略网络 π_θ）**：学习策略，决定"做什么"——给定状态输出动作概率分布
- **Critic（价值网络 V_φ）**：学习价值函数，评价"做得好不好"——输出当前状态的期望价值作为 baseline

### 核心概念速查

| 术语 | 符号 | 含义 |
|------|------|------|
| State（状态） | s_t | 当前时刻的环境观测 |
| Action（动作） | a_t | 智能体在当前状态下的决策 |
| Reward（奖励） | r_t | 环境对动作的即时反馈（标量） |
| Policy（策略） | π(a\|s) | 给定状态 s 下选择动作 a 的概率分布 |
| Return（回报） | G_t | 累积折扣奖励 G_t = Σ_{k=0}^∞ γ^k r_{t+k+1} |
| Value Function | V(s) | V(s) = E[G_t \| s_t=s]，状态 s 下期望回报 |
| Q-Function | Q(s,a) | Q(s,a) = E[G_t \| s_t=s, a_t=a]，执行 a 后的期望回报 |
| Advantage（优势） | A(s,a) | A(s,a) = Q(s,a) - V(s)，动作相对平均水平的优势 |

**优势函数的定义**就是当前状态下做某个动作的回报相比当前状态的期望回报的差值，即表达了这个动作相对于其他动作的优势。直观理解：A>0 比平均水平好，A<0 比平均水平差，A=0 就是平均水平。

## 马尔可夫决策过程

### 马尔可夫性质

下一状态仅取决于当前状态和动作，与历史无关：P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_t, a_t, history)

### 贝尔曼方程

**定义**：当前的即时奖励与下一状态价值的折现组合的期望（广度优先搜索一步的意思，下一步各个概率的乘积和，来估算实际的值）。

```
V(s)   = E[ r + γ V(s')   | s, a ~ π ]
Q(s,a) = E[ r + γ Q(s',a') | s, a, π ]
```

### TD Error（时序差分误差）

蒙特卡洛方法必须要等整个序列结束之后才能计算得到这一次的回报，而时序差分方法只需要当前步结束即可进行计算。具体来说，时序差分算法用当前获得的奖励加上下一个状态的价值估计来作为在当前状态会获得的回报：

```
δ_t = r_t + γ V(s_{t+1}) - V(s_t)
```

**贝尔曼方程和 TD Error 之间的关系**：贝尔曼方程给出了理论上的状态价值关系（当前状态的价值 = 即时奖励 + 下一状态价值的折现组合的期望）。而 TD Error 则是实际获得的即时奖励和下一个状态价值估计与当前状态价值估计之间的差值，可以看作是对贝尔曼方程在实际学习过程中的偏差度量。

**一句话**：贝尔曼方程是"真理"，TD Error 是当前估计与"真理"的差距，是我们更新价值函数的"学习信号"。

- **MC（蒙特卡洛）**：等整个序列结束 → 无偏但方差大，不能用于在线场景
- **TD（时序差分）**：只需一步 → 有偏但方差小，适合在线学习

### 最优策略

在所有状态下，最优策略的状态价值函数大于其他任何策略的状态价值：V^*(s) ≥ V^π(s), ∀s, ∀π

## Reward Model

### ORM（Output Reward Model）

对整个回答的最终结果进行打分（通常在 EOS token 处输出得分）。PPO 和 GRPO 中通常使用 ORM——对一整个句子最后的 EOS 输出的得分。

### PRM（Process Reward Model）

对推理过程中的每一步进行打分，提供更细粒度的奖励信号。适用于需要逐步推理的数学、逻辑任务。

### Reward Model 训练

**数据集**：一个问题 + 两个答案（chosen 和 rejected）。

**Loss 计算**：分别计算 chosen 和 rejected 的得分，使用 logsigmoid 计算值：

```
L = -E[ logsigmoid(r(chosen) - r(rejected)) ]
```

**PairwiseLoss 的良好属性——自带 margin 效果**：因为在 0 附近也会产生一定的 loss，来更新模型，拉大正负例的差距。当 `chosen_reward - reject_reward < 0` 时 loss 急剧上升，表示正例样本得分小于负例样本得分，产生较大 loss，回传产生较大梯度更新模型。当 `chosen_reward - reject_reward > 0` 时 loss 基本趋近于 0，表示正例得分大于负例得分是合理的，不产生 loss。

## On-Policy vs Off-Policy

### On-Policy

**类比**：
```
生成数据和训练都使用同一个模型（策略）。

老师批评和表扬都是根据小明的言行 policy，而小明的言行 policy 也是根据老师的批评改正。
老师根据小明的行为 policy 玩手机，产生批评（生成数据）。
小明根据老师的行为减小玩手机（训练模型）。
```

对于 on-policy 来说，如果策略发生了更新，之前的数据就不能再用于训练，因为这些数据不再符合新策略的分布。换句话说，on-policy 方法要求行为策略和目标策略一致，即 π_behavior = π_target。

**on policy 为什么要丢掉旧数据？** 强化学习的目标是通过不断改进策略来最大化累积奖励。每次策略更新后，智能体的行为会发生变化，从而导致环境中的状态分布也会发生变化。因此，旧数据可能无法准确反映新策略下的环境动态。

**缺点**：由于 on policy 生成的数据只能让模型学习一次，速度慢——需要先生成数据，再去训练。因此产生了 off policy。

### Off-Policy

off policy 通过重要性采样技术，能够重复使用旧的数据，校正旧数据与新策略之间的分布差异。

**类比**：
```
off policy：择其善者而从之，其不善者而改之。

根据老师对小明的批评，如果其他人自己也玩手机的话：
如果自己玩的更多，则需要修改的也更多一些；
如果自己玩的更少，则需要修改的也会少一些。
```

### 重要性采样（Importance Sampling）

**核心公式**：
```
E_{x~p_old} [ f(x) · p_new(x)/p_old(x) ] = E_{x~p_new} [ f(x) ]
```

使用参考策略 θ' 来获取优势价值，再用自己的概率除以参考策略的概率，来判断是否增加或者减小惩罚。这样解决了 on policy 训练效率太低的问题。

**在 LLM 强化学习中的应用**：使用旧数据来在 policy model 判断每一个 step 选择这个 token 的概率，这个概率与 ref model 概率相比的值就是优势值的权重。使用 clip 或者 KL 散度来约束这个权重来确保 ref model 与 policy model 偏差不会变得太大。这里的旧数据就是从 ref model 中采样生成的，后续用于在新的 policy model 中进行训练。因此这也是重要性采样的体现所在——采样的数据是从 ref model 中获得的。

**注意**：only used once——因为在更新参数后，我们的策略已经变了，而先前的数据是基于更新参数前的策略得到的。**off policy 与重要性采样直接相关**。

**风险**：如果实际策略和参考策略的概率分布差距太大，会导致学习不到有用的信息（好学生与坏学生被老师批评的点不一样），因此使用 KL 散度或者 clip 截断函数来限制 p_theta 和 p_theta' 的概率分布。

## PPO

### 四个模型

PPO 训练涉及四个模型（实际通常共享 backbone，仅 head 不同）：

1. **Actor Model（Policy Model）**：策略 π_θ，生成回答并输出每个 token 的 log_prob
2. **Reference Model**：冻结的 SFT 模型（基准模型），用于计算重要性采样权重，限制更新幅度
3. **Reward Model（ORM）**：对整个回答给出标量奖励（在 EOS token 处输出得分）
4. **Critic Model（Value Model）**：针对每一个 step 的状态进行打分，即对当前 token 到最后 EOS 的预估价值

四个模型除了 head 不同，其他都是相同的。因此为了节省显存，对于这些的模型训练，使用同一份模型做冻结，只训练 LoRA 和不同的 head。Actor 和 Critic 共享同一套 LoRA，Reward Model 自己一套 LoRA，各自使用不同的 head。

对于大模型而言，state 就是当前截止的 token 序列，action 就是 next token。

### 重要性采样与重新采样

重采样是对旧策略进行一次采样的旧数据上进行多次的更新。这里同一个 batch 的数据会被使用 n 次，而这里需要注意的是，当完成了一个 epoch 的迭代后，会将新的策略作为旧策略，并且在该策略上重新进行采样。但是通常仅使用一份数据，然后使用 KL div 或者 clip 来限制 policy model 的更新幅度。

**为什么需要重新进行采样？** policy model 更新越来越多之后，新旧策略差距会变得越来越大，这时候重要性采样的作用并不大了。**新旧策略差异变大时，重要性采样的有效性会下降。因此需要使用 KL div 或者 clip 进行约束。**

**总结**：
- **旧数据复用**：在单次迭代内，PPO 允许通过重要性采样多次利用旧策略数据，提升效率
- **重新采样的触发条件**：策略更新次数超过阈值（如一次迭代结束）、新旧策略差异过大导致重要性采样权重方差失控
- **核心结论**：PPO 通过有限复用旧数据平衡效率与稳定性，但必须定期重新采样以确保数据与当前策略分布一致

**Clip 与 KL 散度的互补关系**：

| 机制 | 方式 | 特点 |
|------|------|------|
| Clip | 硬截断概率比率到 [1-ε, 1+ε] | 简单高效，通过截断概率比率实现硬性约束 |
| KL 散度 | 软惩罚偏离程度 | 更平滑，通过衡量分布差异实现软性约束 |
| 共同目标 | 防止新策略过度偏离旧策略 | 平衡策略更新的激进性与稳定性 |

### 优势函数与 GAE（广义优势估计）

**问题**：AE 算法采用 one-step TD 来描述 advantage——TD 算法虽然会减小方差，但是也增大了偏差。MC 方差最大，one-step TD 偏差最大。有没有一种方法，可以提供一种 trade-off，而且可以很方便的调节这种 trade-off 呢？

**GAE 公式**：
```
Â_t^GAE(γ, λ) = Σ_{l=0}^{∞} (γλ)^l δ_{t+l}
其中 δ_t = r_t + γ V(s_{t+1}) - V(s_t)
```

**λ 的作用**：
- λ = 0：只使用一步 TD → 低方差、高偏差
- λ = 1：等价于蒙特卡洛估计 → 高方差、低偏差
- 0 < λ < 1：在偏差和方差之间做 trade-off

**蒙特卡洛方法**：通过对完整的轨迹进行采样，计算实际的回报来估计价值函数。利用从状态 s 到回合结束的总回报来估计 Q(s,a)，然后用 A(s,a) = R(s,a) - V(s)。优点：无偏；缺点：依赖回合完整性、方差大、不能用于在线场景。

**时序差分方法**：通过结合当前时刻的估计值和下一个状态的估计值来更新当前的价值估计。使用 r_t + γV(s_{t+1}) 作为 V(s_t) 的估计量。优点：不需要等待整个回合结束、方差小；缺点：引入了偏差。

### 大模型 PPO 训练流程

```
Pre-train → SFT → Train Reward Model → PPO
```

**Reward Model 训练**：Reward 模型通过输入一个问答对，让模型生成一个得分。数据集为一个问题两个答案（chosen 和 rejected）。Loss 使用 logsigmoid 计算 chosen 和 rejected 的得分差异。

**PPO 目标函数（Clip 版本）**：
```
L^CLIP(θ) = E_t [ min( r_t(θ) A_t,  clip(r_t(θ), 1-ε, 1+ε) A_t ) ]
其中 r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
```
Loss_ppo2：GAE 为正，让 p_theta 尽可能大；当 GAE 为负，让 p_theta 尽可能小。

**Advantage**：计算当前 step 的 return 与 Critic Model 的输出值（label）的差。Return(t) 是当前时间步 t 到最后 reward 值和衰减系数的乘积之和——return 就是 reward 的累积。直观上，总的 return 就是所有 reward 的累积和。

**总奖励公式**：
```
R_total = R_RM - β · D_KL(π_θ || π_ref)
```
1. KL 值用来让训练模型生成 token 的概率分布接近基准模型
2. Reward score 使用 Reward Model 判断该问答的奖励惩罚

**训练伪代码**：
```
1. 用旧策略采样 N 条轨迹
2. 计算 GAE advantage（从后往前遍历）
3. 计算 return = advantage + value
4. 循环 K 个 epoch：
   a. 分 batch 计算新策略的 log_prob
   b. 计算 ratio = exp(new_log_prob - old_log_prob)
   c. 计算 policy loss（clip 版本，取 max 保守更新）
   d. 计算 value loss（也带 clip）
   e. 总 loss = pg_loss + vf_coef * vf_loss → 反向传播
5. 回到步骤 1 重新采样
```

**学后感**：value 值的 label 是一步采样 + 下一步的 value 期望。

## DPO

DPO（Direct Preference Optimization）直接使用偏好数据优化策略，无需显式训练 Reward Model。核心思想是将 RLHF 的 RL 阶段转化为一个简单的分类问题——直接在偏好对上最大化 chosen 相对于 rejected 的概率。

**优势**：不需要训练和维护单独的 Reward Model，不需要 PPO 复杂的四模型架构，训练更稳定、更简单。

---

## GRPO

### 核心哲学

GRPO 是**排序放大器**，而非评分器。

- **本质**：GRPO 优化的是同一 Prompt 下不同 Response 的**相对优势（Advantage）**
- **特性**：对 Reward 绝对数值不敏感，对 **Group 内排序关系**高度敏感
- **金句**："只要排序稳，训练就能滚；排序一乱，模型必散。"

### 核心创新点

1. **去掉 Critic 网络**：直接用 Group 内平均 Reward 作为 baseline
   - `Advantage_i = R_i - mean(R_1, ..., R_G)`
2. **Reference 模型约束**：用冻结 SFT 模型计算 KL 散度惩罚
3. **简化架构**：减少训练目标、超参数和显存开销

**为什么去掉 Critic？** 简化架构，减少训练目标和超参数；适合大模型 RLHF 场景，Reward Model 已经提供了足够信号；节省显存和计算资源。

**GRPO 作为一个 on-policy 方法，使用重要性采样（Reference Model）的一个原因是让训练更加稳定。如果不使用 Ref Model，模型更新幅度太大，容易训歪，不收敛。**

### Loss 函数

```
L_GRPO = (1/G) Σ_i [ -min( r_i(θ) Â_i,  clip(r_i(θ), 1-ε, 1+ε) Â_i ) + β · D_KL(π_θ || π_ref) ]
```

其中 r_i(θ) = π_θ(o_i|q) / π_ref(o_i|q) 是重要性采样比率，Â_i 是 Group 内归一化后的优势（减去组内均值再除以组内标准差）。

### 训练初期 Loss 从 0 开始上升的原因

- 开始时两个模型相等，重要性采样这部分参数为 1，同时相对优势的加和为 0，所以前一项为 0
- 还是因为初始两个模型是相同的，所以 KL 散度为 0
- Loss 上升是因为 Actor Model 偏离了 Ref Model，所以 Loss 会上升。因为 KL 散度变大，表示模型在探索，并且这个探索的程度比 Reward 奖励的分数要大，抵消了，因此这个 loss 是在上升的。

### GRPO 超参数调优速查

| 参数 | 作用 | 调优建议 |
|------|------|---------|
| `rollout_batch_size` | 每次生成阶段的 prompt 总数 | 显存不足时减小 |
| `ppo_micro_batch_size_per_gpu` | 每 GPU 每步处理样本数 | 显存不足时 40→24，充裕时增大 |
| `ppo_mini_batch_size` | 一次参数更新用多少样本 | 需与 micro_batch × GPU 整除 |
| `rollout.n` | 每个 prompt 生成几个回答（即 G） | 减小加快训练，增大增加多样性 |
| `kl_loss_coef` | KL 惩罚系数 β | KL 飙升调大，Reward 停滞调小 |
| `gpu_memory_utilization` | vLLM 显存占比 | 显存充裕时提高（0.6→0.8） |

### 梯度累积机制

1. 框架读取 mini-batch（如 256 样本）
2. 将其拆成多轮 micro-batch（每 GPU 处理 `ppo_micro_batch_size_per_gpu` 个样本）
3. 每一轮执行前向、反向（`loss.backward()`），但不做 `optimizer.step()`，而累积梯度
4. 当累计的样本数 ≥ mini-batch 阈值时，执行 `optimizer.step()` 并清空梯度
5. 继续下一轮 mini-batch

这种方式允许用更小内存完成训练，但等效于较大 batch size 的训练行为。

### 实践调整建议

**A. 若显存不足**：降低 `ppo_micro_batch_size_per_gpu`（如 40→24）、开启 FSDP 参数 offload、保持 `use_remove_padding=True`、启用 `enable_gradient_checkpointing=True`

**B. 若希望提升训练速度**：增加 `ppo_micro_batch_size_per_gpu`（前提显存允许）、适当增加 mini-batch（如 256→320 或 384）、减小 `rollout.n`（减少生成样本数）、提升 vLLM 的 `gpu_memory_utilization`（如 0.6→0.8）

**C. 保持训练质量与稳定性**：mini-batch 大小与 micro-batch × GPU 尽量整除、调节 `kl_loss_coef` 和 `lr` 在 batch size 改变时配套调整、监控 `loss` 和 `throughput` 日志

## KL 散度在 GRPO 中的核心作用

### 数学本质：从交叉熵到 KL 惩罚

在语言模型训练中，我们通常处理的是概率分布。

- **交叉熵（Cross Entropy）**：`H(P, Q) = -Σ P(x) log Q(x)`。在 SFT 阶段，我们最小化交叉熵，本质是让模型 Q 的输出尽可能接近标签分布 P。
- **KL 散度**：`D_KL(P || Q) = Σ P(x) log(P(x)/Q(x))`。它衡量的是两个分布之间的"距离"或信息损失。

### 在 GRPO 公式中的关系

GRPO 的 Loss 函数中包含了一个显式的 KL 惩罚项：

```
Loss ≈ (1/G) Σ [ Advantage_i · ∇ log π_θ(o_i|q) - β · D_KL(π_θ || π_ref) ]
```

这里 π_θ 是当前正在训练的策略，π_ref 是冻结的参考模型（通常是 SFT 后的模型）。

**核心逻辑**：
1. **交叉熵是"向谁看齐"**：SFT 让模型学会说话的格式
2. **KL 是"别跑太远"**：GRPO 允许模型为了高 Reward 去探索，但 KL 项会像一根橡皮筋，把模型往 π_ref 拉，防止它为了刷分而变成无意义的符号序列（Reward Hacking）

### 为什么 KL 漂移会导致训练崩溃？

在 GRPO 中，如果没有足够强的 KL 约束，会出现以下连锁反应：

1. **策略偏移（Policy Drift）**：模型发现某个特定的 Token 序列（比如重复某个标点或固定格式）能骗过 Reward Model 拿到高分
2. **正反馈循环**：由于 GRPO 放大 Advantage，这个"作弊"路径会被迅速强化
3. **分布塌缩**：此时模型不再根据语义生成，而是进入了 Reward Model 的盲区。表现为 KL 散度突然从正常的 2.0-5.0 飙升到 20+ 甚至更高
4. **梯度爆炸**：当 π_θ 与 π_ref 差异过大时，梯度方向变得极度不稳定，导致 Loss 无法收敛，模型直接"崩了"（输出乱码或复读）

### GRPO 中的 KL 计算：在线 vs 离线

在 PPO 中，通常使用一个可学习的偏置来处理 KL；而在 GRPO 工程实践中，为了效率，通常采用以下两种方式：

- **Token-level KL（精确但慢）**：计算每一个 Token 在新旧模型下的概率比

**Approximate KL（常用）**：
```
D_KL ≈ log(π_θ / π_ref)
```

在训练日志中，如果你看到这个数值持续上升，说明模型正在"变坏"。

### 如何调优 KL 参数 β？

这是体现工程经验的地方。

| 阶段 | β 设置 | 目标 |
|------|-------|------|
| Warm-up 初期 | 较大 | 保持表达能力，微调排序 |
| 中期探索 | 适当减小 | 释放探索空间，追求更高 Reward |
| 崩溃预警 | 立即增大 | 当 Per-token KL > 0.2~0.3 时"踩刹车" |

### CE vs KL 总结对比

| 维度 | 交叉熵（CE） | KL 散度（KL） |
|------|------------|-------------|
| 阶段 | SFT（监督微调） | RLHF / GRPO（强化学习） |
| 目标 | 模仿：让模型学会"正确答案" | 约束：让模型在优化 Reward 时不忘本 |
| 作用 | 决定了模型能力的底色 | 决定了训练的稳定性和多样性 |
| 崩溃表现 | 过拟合（Loss 过低） | 策略塌缩 / 复读机（KL 飙升） |

### 面试：KL 飙升怎么办？

**面试官**："在 GRPO 训练过程中，你提到过 KL 约束非常重要。那在实际看 Tensorboard 曲线时，如果你发现 Reward 在涨，但 KL 散度也在剧烈飙升，甚至超过了 20-30，这时候你会怎么判断？你会调整哪些参数？"

**💡 高分回答话术**：
"这是一个非常经典的 **Reward Hacking（奖励欺骗）** 预警信号。

**首先，我的判断是**：此时模型并不是真的变聪明了，而是在'投机取巧'。当 KL 飙升伴随 Reward 上涨时，通常意味着模型发现了一些 π_ref（参考模型）概率极低、但 Reward Model 评分极高的'偏门'路径（比如特定的格式、冗余的礼貌用语或复读）。

**其次，我会观察两个关键指标**：
1. **Per-token KL**：如果平均每个 token 的 KL 超过了 0.2 或 0.3，说明模型正在丧失原本的语言分布
2. **Response Length**：检查长度是否异常增长，因为 Reward Model 经常对长文本有正向偏见

**针对性的解决措施我会采取三步**：
1. **加强约束**：立即调大 KL 惩罚系数 β。如果是在训练初期，我会使用一个较长的 KL warm-up 阶段
2. **重标定 Reward**：检查 Reward 是否有明显的 Outlier。我会引入 **Rank-based Reward** 或者对 Reward 进行 **Z-score 归一化**，减小异常高分对梯度的冲击
3. **引入参考模型采样**：如果模型已经崩了，我会回滚到上一个稳定的 Checkpoint，并增加训练时的 **Entropy Bonus（熵奖励）**，强制模型保持输出的多样性，防止它锁死在那个高 KL 的错误路径上。"

## Group Size 与采样策略

在 GRPO 训练中，**Group Size（G）** 是一个既关乎"算法收敛"又关乎"算力显存"的核心超参数。它直接决定了你的 Advantage（优势值）计算是否准确。

### 核心逻辑：Group Size 为何至关重要？

在 PPO 中，我们通过一个单独的 Value Network（Critic）来估算基准线；而在 GRPO 中，我们通过 **Group 内的平均 Reward** 来充当基准线。

- **G 太小（如 4 或 8）**：基准线不稳定。如果这 4 个样本都很烂，其中"稍好一点"的烂样本也会被给以巨大的正向 Advantage，导致模型学到错误的方向
- **G 足够大（如 32、64 或更高）**：能够更准确地逼近当前策略的期望回报 E[R]，减小 Advantage 的方差，训练更稳

### 采样多样性——GRPO 的"生命线"

如果 Group Size 很大，但采样出的 G 个回复语义高度重复（温度 T 太低），GRPO 就会失效。

- **采样温度（T）**：通常设置在 **0.7 - 1.0**。高温度能确保 Group 内既有"满分作文"，也有"离题万里"，这种强烈的对比是梯度下降的动力
- **Top-P / Min-P**：配合使用，防止高温度导致输出完全乱码

### 工程平衡：显存 vs. 稳定性

在分布式训练中，增大 G 会直接导致显存爆炸（因为要存 G 倍的激活值）。

| 策略 | 描述 | 优点 |
|------|------|------|
| Micro-batching | 将 G 个样本分批次在前向传播中计算 | 节省显存，支持大 Group Size |
| Reference Cache | 提前计算并缓存 π_ref 的 log_probs | 减少一半的推理计算量 |
| Selective Activation Checkpointing | 只对部分层做检查点 | 在速度和显存间取得平衡 |

### 如何选择 Group Size（面试话术）

**面试官**："你在训练时，Group Size 一般选多大？如果显存不够，你会怎么折中？"

**回答**："在我的经验中，Group Size 的选择取决于**任务的复杂度**。对于数学或代码等有明确判别准则的任务，**16 到 32** 通常是性价比最高的区间；如果是开放性对话，可能需要 **64** 以上才能获得稳定的基准线。

如果面临显存压力，我不会直接减小 G，因为这会损害训练稳定性。我会优先采取以下工程手段：
1. **梯度累积（Gradient Accumulation）**：保持大的全局 Batch Size
2. **推理分发**：采样阶段（Sampling）和训练阶段（Training）解耦，利用推理引擎（如 vLLM）快速生成样本，再回传给训练框架
3. **计算优化**：采用 **Packing 策略**，将不同长度的回复拼接在一起减少 Padding 浪费，从而在同等显存下容纳更大的 G。"

### GRPO 采样"三剑客"

1. **高 G（Group Size）**：保证 Advantage 的**准确性**
2. **高 T（Temperature）**：保证 Group 内的**对比度**
3. **强 KL**：保证探索不离谱的**安全性**

## 数据侧与 Hard Cases 构造

这是 GRPO 训练中最进阶的部分。很多人认为 RL 只是在练模型，其实 **RL 练的是"数据差"**。如果 SFT 数据是告诉模型"什么是对的"，那么 RL 的数据（Prompt）则是为了逼出模型"容易犯错的地方"。

### 为什么不能直接搬 SFT 的数据？

SFT 的数据通常是"由易到难"且带标准答案的。但在 GRPO 阶段，如果 Prompt 太简单，会出现以下问题：

- **无区分度**：G 个样本全对，Advantage 全是 0，模型学不到东西
- **奖励饱和**：模型已经满分了，KL 散度却在白白上升

**结论**：RL 需要的是 **"临界区数据"**——即模型处于"跳一跳够得着"或"容易掉坑里"的题目。

### 如何构造高质量的 RL Prompt（Hard Cases）？

**A. 挖掘模型弱点（Error Mining）**
- 用现有的 SFT 模型跑一遍题库，把那些 **Pass@1 低但 Pass@10 高** 的题目挑出来
- Pass@1 低说明模型会犯错；Pass@10 高说明模型有潜力做对。GRPO 的目标就是通过强化，把 Pass@10 的潜力转化为 Pass@1 的能力

**B. 增加约束复杂度（Constraint Following）**
在原始问题上人为增加限制条件，强制模型进行深思：
- 数学："不准使用解析几何方法，只能用纯代数推导"
- 代码："要求时间复杂度 O(n)，且不能调用第三方库"
- 文案："回复字数必须在 150-200 字之间，且每段开头必须是动词"

**C. 对抗性 Prompt（Adversarial Prompts）**
故意引导模型进入逻辑陷阱（例如："为什么 1.11 比 1.9 大？"），训练模型在面对误导时仍能保持逻辑连贯。

### 数据配比的"黄金比例"

在实际工程中，我们不会只投喂难题。通常采用**混合策略**：
1. **30% 基础题**：防止模型退化，保住 SFT 的基本盘
2. **50% 挑战题**：核心涨分点，通过多采样（G）形成好坏对比
3. **20% 判别/纠错题**：给模型一个错误的回答，让它指出错误并修正（这是提升逻辑思维上限的杀手锏）

### 如何评估 RL 数据的质量？

**面试官**："如果你发现 GRPO 训练一段时间后效果不再提升，但 Reward 已经很高了，除了调参，你会怎么动数据？"

**回答**："我会从 **'数据有效性'** 的角度进行排查和迭代：
1. **检测'无效 Group'**：统计训练日志中 Advantage 为 0 的样本占比。如果大部分样本在 Group 内得分一致（全对或全错），说明这批数据对当前模型太简单或太难。引入更具挑战性的 **Hard Cases** 来恢复区分度。
2. **引入 Cross-validation**：用一个更强的模型（如 GPT-4o 或更大的开源模型）对当前 GRPO 产出的 Top 样本进行评价。如果 RM 给高分但强模型给低分，说明出现了 **Reward Hacking**，针对性地加入反面教材（Negative Samples）重训 RM。
3. **动态调整难度**：采用类似 **'课程学习'** 的方法。随着训练步数增加，逐步提高 Prompt 的约束复杂度和逻辑链条长度，确保模型始终处于'非舒适区'进行学习。"

### GRPO 数据的三个指标

- **区分度（Discrimination）**：Group 内一定要有对有错
- **覆盖度（Coverage）**：场景要广，防止模型在单一任务上过拟合
- **真实性（Faithfulness）**：拒绝 RM 刷分，确保高分对应的是真实的逻辑提升

## GRPO 的工程问题与对策

### 问题 1：Reward 失效与重标定

**场景**：Policy 提升后，Reward 失去区分度（均值高、方差极小）。

| 方案 | 核心逻辑 | 适用场景 |
|------|---------|---------|
| Z-Score | 对 Group 内 Reward 标准化 | 放大微小差异，防止梯度被 KL 淹没 |
| Rank-based | 仅保留排序（1,2,3...），丢弃分值 | **最推荐**，根本上解决饱和与 Outlier |
| Binary Win/Lose | 只分 Top-1 和非 Top-1 | Reward 噪声大、仅"最好"结果可靠时 |
| Margin-based | 人为拉开样本间 Reward 间隔 | 训练停滞，需要强化探索时 |

**进阶对策**：课程学习（由易到难）、周期性增量更新 RM、增加辅助 Reward（长度、多样性控制）。

### 问题 2：训练崩溃

**根因**：相对信号被噪声放大。Outlier 主导梯度，导致策略瞬时发散。

**典型征兆**：KL 散度突然飙升、Top-1 输出开始模板化（复读机前兆）、Reward 持续上升但人工评估质量（GA0）下降（Reward Hacking）。

**最小应对集合**：
- 约束：强 KL 惩罚（初期必须稳）+ 梯度裁剪（必备）
- 采样：增大 Group Size（提供更稳基准）+ 提高温度（增加多样性）
- 优化：学习率随 Group Size 增大而适度调低

### 问题 3：复读机现象（Mode Collapse）

**根因**：策略塌缩。模型发现"重复、安全"的模板能稳定获得 Group 内的高分。

**三部曲解法**：
1. **Reward 层**：引入显式惩罚（n-gram repetition penalty）或多样性奖励（Entropy bonus）
2. **数据层**：使用 Hard Prompt 或 Adversarial Prompt，增加采样随机性
3. **算法层**：增大 Group Size 以打破局部最优，加强 KL 约束防止偏离原模型太远

### 问题 4：奖励崩塌（Reward Collapse）

**深层原因分析**：

1. **舍弃 Critic 网络导致梯度估计方差增大**：GRPO 采用"Actor-only"架构，去掉了传统 Actor-Critic（AC）架构中的 Critic 网络。Critic 的作用是通过值函数估计（如 Advantage 函数）为策略梯度提供稳定基准，而 GRPO 改为在线估计 Advantage 函数，这种简化虽节省存储，但导致梯度估计的方差显著增大。方差增大会使策略更新方向偏离最优路径，从而引发奖励值剧烈波动甚至骤降。

2. **小批量数据加剧不稳定性**：当训练批量（batch size）较小时，策略梯度的估计误差会进一步放大。GRPO 依赖大量数据来降低方差（通过"时间换空间"的策略），但在中小规模团队或资源受限场景下，数据量不足会导致梯度计算不稳定。

3. **在线估计的局限性**：GRPO 通过组内相对奖励计算 Advantage 函数，避免了训练额外价值模型，但这种在线估计方法对数据分布敏感。若一次采样中的有效奖励值稀疏（如某些复杂任务中正样本较少），估计的 Advantage 函数会失真，导致策略更新方向错误，进而引发 reward 崩塌。

4. **与 PPO 等算法的对比劣势**：相比之下，PPO 等算法通过显式训练 Critic 网络（如价值函数模型）来稳定 Advantage 估计，尽管需额外计算资源，但能显著减少方差。而 GRPO 为节省存储选择"去 Critic 化"，在资源有限时需以牺牲稳定性为代价。

**总结来看**，GRPO 的 reward 崩塌本质是算法设计在存储优化与稳定性之间的权衡失败，尤其在数据稀疏或资源受限场景下更为明显。

**DeepSeek 的成功例外**：DeepSeek 能成功应用 GRPO，主要依赖于其海量数据支撑。足够大的批量可抵消方差问题，使策略迭代趋于稳定。但对于大多数缺乏数据和算力的团队，GRPO 的稳定性缺陷难以规避，易出现 reward 崩塌。

**本质原因**：缺少了 Critic 网络提供的指导方向。因为 reward 大小是根据 rollout 的样本计算的，当 rollout 的质量不好时，会使策略往奇怪的方向移动，这也是导致策略梯度方差变大的原因。同时奖励计算对数据分布敏感，这也是 GRPO 训练不稳定的原因。

### 问题 5：奖励震荡与长度剥削

**奖励震荡**：由于 max_completion_length 的设置，当模型思考比较长，而被截断没有生成最终的答案时，Reward Model 可能因为无法获取到最终答案，而给当前回答打低分，即使当前推理的行为是合理的。因此模型误以为生成长度过长而被惩罚，导致模型生成的长度又变短。

**长度剥削机制**：
- **max_completion_length 过小**：合理的推理过程因长度被截断而被打低分 → 模型误以为长推理不好 → 长度骤降
- **max_completion_length 过大**：结果导向的 reward 会剥削模型长度，这种剥削长度是一种 reward hacking。模型倾向于尽量剥削输出长度来达到更高的 reward 而不理会中间的胡言乱语过程，从而造成胡言乱语现象的加剧，进而导致输出长度进一步提升

**震荡循环**：输出长度快速上涨 → 部分输出未能在规定窗口内给出正确答案 → 训练 reward 开始下降 → 模型输出又突然整体变短以提升训练 reward → 输出长度振荡

**最佳实践**：分阶段增加训练窗口，让模型逐级适配更长的输出长度以及长窗口下的长文注意力。训练窗口长输出长度上涨更明显，但窗口也不能过长——太长的窗口训练效率低且容易产生 length hacking。较好的做法是逐步增加窗口，逐步适配。

## DAPO

### 四大核心改进

**1. Clip-Shifting（裁剪偏移）**

**问题**：随着训练的进行，策略的熵迅速降低，导致某些 group 采样出的 response 几乎相同。有限的探索和过早确定性的策略，可能会阻碍扩展过程。

**思想**：对于上裁剪的阈值，更高概率的 token，因域值的原因，变化程度能够更大（比如 0.9 × 1.2 = 1.08），而低概率 token 变化小（0.01 × 1.2 = 0.012）。能够提高低概率 token 的探索空间——一般 low 取小点的值，high 取大的值。防止高概率变化大导致分布变得太低，帮助低概率分布上升更大利于探索。

**效果**：Clip 机制使模型保持在一个熵比较高的状态，使在训练多个 step 后依然在探索，而不会过早确定值，使得陷入局部最小值。

**2. Dynamic Sampling（动态采样）**

**问题**：后期零优势。对于 GRPO，如果特定提示的所有输出都是正确的，并且获得相同的奖励 1，则该组的最终优势为零。零优势导致策略更新没有梯度，从而降低样本效率。

**方案**：候选样本修正——将批次中的所有提示保留为有效梯度，并保持提示数量一致。在训练之前，继续采样，直到该批次完全填满准确率既不为 0 也不为 1 的样本（正确的样本要小于 G 也要大于 0）。

**效果**：动态采样使模型训练更高效，更早达到峰值。主要还是由于普通的 GRPO 到后期模型比较难生成差的样本，导致奖励都为 0。

**3. Token-Level Policy Gradient Loss**

**问题**：GRPO 使用的是 seq level——先对句子内部求平均，再对不同长度的 rollout 加入相同的长度求平均。这导致长文本中每个 token 的 loss 权重相对更低。

**具体问题**：
- 对于高质量的长样本，这种效应可能阻碍模型学习其中的推理相关模式
- 过长的样本通常表现出低质量模式（如胡言乱语和重复词）
- 样本级损失计算由于无法有效惩罚长样本中的这些不良模式，导致熵和响应长度不健康增加

**DAPO 改进**：统计所有 rollout 的总 token 数，对所有 token 的 loss 求平均，与 sample 无关。通俗来说就是修改了求和求平均的顺序——原始 GRPO 公式是每个采样结果计算平均 token 损失，再计算样本平均损失（长样本 token 被"稀释"）；DAPO 将所有样本的所有 token 损失总和除以所有 token 数，loss 由长样本来主导。

**为什么较长回复中的 Token 对总体损失的贡献更低？** 因为每个样本都被平均到 Token 级别了，和长度无关了，导致长样本的影响被"稀释"了，对总 Loss 的贡献和短样本相同。改进部分体现在损失函数的求和部分——较长的序列对整体 Loss 的影响更大。此外，从单个 Token 的角度来看，如果某种特定的生成模式能够导致奖励的增加或减少，那么无论该模式出现在多长的回复中，它都会被同等地促进或抑制。

**4. Overlong Reward Shaping**

**问题**：在 RL 训练中，通常为生成设置最大长度，过长的回复会相应截断。论文发现，截断样本的不当奖励塑造（一般提取不到答案，所以 Reward 为 -1）会引入奖励噪声，并严重扰乱训练过程。因为一个合理的推理过程可能仅因长度过长而受到惩罚，使模型对其推理过程的有效性产生困惑。

**方案**：采用过长样本过滤（Overlong Filtering）策略，对被截断的样本屏蔽其损失。此方法能显著稳定训练并提升模型性能。

**更先进的方案——缓存区机制主动预防超长**：针对不同长度加入了自适应的长度惩罚，而不使用截断的方式。能够使模型主动学习不过长的采样结果。该惩罚将被添加到原始的基于规则的正确性奖励中，从而引导模型避免生成过长的响应。

**个人对于长度的理解**：
- **句子过长被动截断**：导致模型的合理的推理思考分布只是因最终答案被截断，导致模型误以为这样长的合理的分布是不好的，导致 reward 降低。这也被称为 **reward collapse due to truncation bias**
- **主动预防**：把长度过长加入到 reward 中，而不是因为提取不到答案认为这样的分布是有害的来减少合理的分布

## GSPO

### GRPO 的两大局限

1. **重要性采样方差问题**：重要性采样的原理是重复采样多次求平均来获得重要性权重的，而 GRPO 中仅凭一个词来作为权重，这是不符合原理的并且方差比较大。仅采样一个 token，这个权重充满噪声。并且由于一个序列中，文本前后是有一定关联的，那么这样的方式导致最后权重的偏差像滚雪球一样越来越大，分布偏移比较远。随着句子变得越长，噪声累积越大。

   因为 **GRPO 在 token 级别使用单一样本来估计重要性权重，而整个序列的更新目标是这些 token 级 log-prob 的求和**。当每个 token 的重要性权重都带有随机噪声时，**这些噪声不会相互抵消，而是被逐 token 相加，导致方差随序列长度近似线性甚至指数级增长**，从而引发训练不稳定。

2. **MoE 模型路由回放问题**：对于 MoE 模型来说，模型更新前后同一个 token 路由的专家不同。因此加入了路由回放这种方式，使训练系统更加复杂。

### GSPO 的改进

**1. 奖励与重要性权重粒度对齐**：对所有的重要性权重相乘开几何根，获取整个序列的重要性权重。它直观地反映了：对于旧模型生成的这个回答，我们的新模型是更"喜欢"它了（概率变高），还是更"讨厌"它了（概率变低）。这才是对重要性采样的正确应用。

**梯度对比：GSPO 为何更稳定？**

- **GRPO 的指挥信号**：对于一个好回答，它会对里面的每个词说："A 词，你很重要，给你 1.2 倍的权重去学习！B 词，你一般般，给你 0.9 倍的权重"……这些权重充满噪声，指令互相矛盾，模型学起来晕头转向。
- **GSPO 的指挥信号**：对于一个好回答，它会先给出一个总体的评价，比如"这个回答整体不错，权重是 1.1"，然后对里面的所有词说："大家是一个团队，都朝着这个 1.1 倍权重的方向调整。"

**2. 没有 MoE 训练的路由回放问题**：因为它关注的是宏观的、整个序列的概率，这个指标对于底层具体的专家组合变化不那么敏感。GSPO 无需任何额外技巧，就能稳定地训练 MoE 模型，大大简化了训练流程。

**3. GSPO 裁减率更高**：这让模型能够使用更少的数据，但是效果比 GRPO 更好。原因是 GRPO 上没有被裁减的一些噪声大的数据，把好的数据的微小提升给抵消了。这说明**学习信号的质量远比数量重要。**

## SAPO

> Paper：Softmax Policy Optimization with Adaptive Clipping

### Motivation

| 方法 | 核心逻辑 | 硬裁剪的问题 |
|------|---------|------------|
| GRPO | token 级硬裁剪：超出 [1-ε, 1+ε] 范围的 token 直接归零梯度 | "全有或全无"：有效信号与噪声一同被截断，样本效率低；梯度不连续，易导致训练震荡或崩溃；无法平衡"抑制噪声"与"保留有用更新" |
| GSPO | 序列级硬裁剪：对整个序列的平均比率裁剪 | 序列内 token 同质化假设：若序列中存在少量极端 off-policy token，会导致整个序列的梯度被抑制，浪费大量 near-on-policy token 的有效信号；硬裁剪带的脆性：裁剪阈值（ε）难以调优，过紧或过松均影响性能 |

**一个遗憾**：两者均未考虑"正负优势 token（positive/negative Â_i,t）的稳定性差异"——负优势 token 的梯度易扩散到大量无关 token，是训练不稳定的重要来源，但 GSPO/GRPO 对正负 token 采用相同处理逻辑。

### Method

**核心思路**：用"平滑自适应的软门控"替代"非连续的硬裁剪"，同时通过"序列 - token 双粒度适配"和"不对称温度控制"，兼顾稳定性、样本效率和适应性。

**改进 1：硬裁剪 → 温度控制的软门控（核心突破）**

SAPO 用 sigmoid 函数构建的软门控替代 GSPO/GRPO 的硬裁剪，实现梯度的**平滑衰减**而非"断崖式截断"。

软门控公式：`g(r; T) = sigmoid(T · (1 - |r - 1|))`，其中 r 是重要性采样比率，T 是温度参数。当 |r-1|=0 时 g=1（完全通过），|r-1| 增大时平滑衰减。

**改进 2：单一粒度适配 → 序列 - token 双粒度自适应**

SAPO 同时实现"序列级连贯性"和"token 级灵活性"，弥补 GSPO（仅序列级）和 GRPO（仅 token 级）的片面性：
- 当每个 token 的重要性比例在 1 附近且离散程度低时，SAPO 的 token 级软门控平均值收敛为"序列级软门控"，行为与 GSPO 一致
- token 级灵活性：当序列中存在少量极端 off-policy token 时，SAPO 仅对异常 token 降权，保留其他 near-on-policy token 的有效信号，大幅提升样本效率

**改进 3：统一温度 → 不对称温度控制**

SAPO 首次引入"正负优势 token 的差异化温度设置"，解决负 token 梯度导致的不稳定性：
- 负优势 token 会提升大量无关 token 的 logits，梯度扩散更严重，稳定性风险更高 → 需要让负 token 的梯度衰减更快
- 温度配置：T_负 > T_正（负 token 温度更高），使得负 token 的梯度权重衰减速率显著快于正 token
- 实证验证：若 T_负 < T_正，会出现明显训练崩溃

### 差异总结

| 对比维度 | GRPO（token 级硬裁剪） | GSPO（序列级硬裁剪） | SAPO（软门控 + 双粒度 + 不对称温度） |
|---------|---------------------|---------------------|----------------------------------|
| 裁剪/门控方式 | 硬裁剪（超出范围梯度归零） | 硬裁剪（序列比率超出抑制全序列） | 软门控（平滑衰减，无梯度归零） |
| 粒度适配 | 仅 token 级 | 仅序列级 | 序列 + token 双粒度自适应 |
| 正负 token 处理 | 无差异 | 无差异 | 不对称温度（T_负 > T_正） |
| 样本效率 | 低（有效 token 易被截断） | 低（异常 token 导致全序列信号浪费） | 高（仅降权异常 token，保留有效信号） |
| 训练稳定性 | 差（梯度不连续，易震荡） | 中等（序列级裁剪减少方差，但脆性强） | 高（平滑梯度 + 负 token 抑制，不易崩溃） |

SAPO 的本质是通过"平滑化"和"自适应化"解决硬裁剪的固有缺陷：对 GSPO 保留序列级连贯性同时解决异常 token 浪费全序列信号的问题，对 GRPO 保留 token 级灵活性同时解决硬裁剪导致的梯度不连续和样本效率低的问题。最终，SAPO 为 LLM 的 RL 训练提供了更可靠、更具扩展性的优化策略，尤其适用于 MoE 模型（token 比率方差更高）和多模态任务（信号异质性强）。

## RFT 与 ReFT

**RFT（Rejection Fine-Tuning，OpenAI）** 和 **ReFT（Reinforced Fine-Tuning，ByteDance）** 的核心思想：

**"只要能定制好一个任务的 verifier，那么 RFT 便可以在这个新的领域场景，以十分之一或更少的数据，轻松超过 SFT 的结果。"**

**流程**：
1. 使用某些方法（拒采样、MCTS、知识数据蒸馏）得到推理样本，对模型进行 warmup
2. 直接进行 online reinforcement learning
3. 强化学习中使用原始 QA 对进行训练
4. 使用最后的输出与 ground truth 进行对比得到 reward
5. **不需要进行人类反馈**——这是与 RLHF 的关键区别

**与 RLHF 的区别**：RLHF 需要训练 Reward Model 来模拟人类偏好，而 RFT/ReFT 直接用 verifier（如数学题的答案正确性检查）作为 reward，省去了人类反馈和 Reward Model 训练。

## 熵坍塌问题与解决方案

### 熵坍塌现象

随着训练的进行，entropy 逐渐降低，导致某些 group 采样出的 response 几乎相同，使得模型在早期变得更加确定，限制了模型的探索空间。

**典型现象**：
- 前 200 步：熵下降 73%，模型性能提升 76%
- 前 800 步：熵下降 94%，模型性能提升 93%
- **后 1600 步：熵仅下降 6%，模型性能仅提升 7%**

得出结论：模型在训练早期（少量步数）就决定了最终性能。

### 方案 1：Clip-Higher

**核心思想**：低概率的倍数增加小，难以增大，因此把上限提高，能够让低概率的 token 能够进一步探索。Clip-Higher 让大概率的 token 减速下降，让小概率的 token 加速上升。因为大概率改变的幅度会很大，所以下降时需要刹车。

**具体做法**：设置 clip 范围为 [low, high]，其中 low < 1，high > 1，且 |low-1| < |high-1|。例如 [0.8, 1.5] 而不是对称的 [0.5, 1.5] 或 [0.8, 1.2]。

**效果**：能够提高低概率 token 的探索空间。防止高概率变化大分布变得太低，帮助低概率分布上升更大利于探索。最后的结果是训练的熵不会一下子崩塌，也说明模型其实依然在探索。

### 方案 2：Adaptive Entropy Control

**目标**：动态调节熵损失系数 α_k，使当前模型的熵不低于目标熵（tgt-ent）

**超参数**：tgt-ent（期望维持的目标熵，如 0.2）、Δ（每步允许调整的熵系数变化幅度）

**算法流程**：
1. 初始化 c_0 = 0
2. 每个 rollout step 动态调节：if 当前熵 e < tgt-ent → c_{k+1} = c_k + Δ；if e > tgt-ent → c_{k+1} = c_k - Δ

**表达式**：L_total = L_RL + max(c_k, 0) · (-H(π_θ))

即：只有当当前熵低于目标熵时，才启用熵项进行正则化。实验中将目标熵设置为 0.2，模型在后期的熵始终在 0.2 左右波动。

### 方案 3：Clip-Cov / KL-Cov

**核心洞察**：**熵降低 → 策略变得过于自信 → 探索能力下降 → 下游性能停滞**

**高概率 + 高 advantage 的动作会加剧熵的下降；低概率 + 高 advantage 的动作可提升熵。**

```
协方差 Cov(p, A) > 0 → 策略与奖励一致性越强，熵下降越快
协方差 Cov(p, A) > 0 → 强化高概率、高回报 token，策略更加自信
协方差 Cov(p, A) < 0 → 强化低概率、高回报 token，策略进入探索
```

**关键发现**：策略熵下降（entropy collapse）主要由极少数高协方差 token 导致，因此只需限制它们的更新即可防止熵坍塌。仅**前 0.02%** 的 token 协方差值比平均值高出 **近 2000 倍**，可见熵下降被极少 token 主导。

针对上述现象，作者通过**限制高协方差 token 的更新幅度**来防止熵坍塌，提出了两种方法：
- **Clip-Cov**：随机裁剪掉高协方差的 token（即 detach 它们的梯度，不参与策略更新）
- **KL-Cov**：对于协方差最高的少数 token，加入 KL 正则项控制其更新强度

**Clip-Cov vs Clip-Higher**：

| 维度 | Clip-Higher | Clip/KL-Cov |
|------|------------|-------------|
| 干预对象 | 所有低概率 token | 仅 0.02% 高协方差 token |
| 理论依据 | 启发式 | 协方差公式推导 |
| 稳定性 | 后期性能震荡 | 持续提升 |
| 控制精度 | 间接影响熵 | 直接调控熵动态 |

## 强化学习 QA

### 为什么 GRPO 中 rA 部分从 0 开始变大？

**1. 策略更新导致生成分布拉开**
- 初始时 π_θ ≈ π_ref，生成内容差异小 → reward 集中
- 随着训练，PPO 部分会推动模型更倾向于高 reward 输出，同时低 reward 输出会因为探索仍然出现
- 这样一来，样本 reward 之间的差距就被拉大，方差自然增大
  - 举例：一开始所有输出 reward ≈ 0.5，后来一些样本提升到 0.8，而另一些仍在 0.3~0.4 → 方差增加

**2. Reward model 本身是判别式的**
- reward model 多数是 pairwise 训练出来的，它在"差不多的答案"上打分接近，但在"明显对/错"的答案上差距大
- 当策略在训练后开始探索到一些更优的模式（比如答案接近正确解）和一些无效模式（走偏了），reward model 就会给出强烈区分
- 这使得 reward 分布由"集中一团"变成"两极分化"，方差增大

**3. Advantage 基于批次均值放大差异**
- Advantage 的计算方式是：A_t = r_t - r̄（r̄ 是 batch reward 均值）
- 如果 batch 内 reward 分布比较平（大家都差不多），advantage 小
- 但当训练后，batch 内出现"好样本拉高均值 + 坏样本明显落后"，advantage 的正负差异会更明显 → 方差增大

**4. 探索与多样性**
- 在采样时通常会加入温度/Top-p/Top-k，保证一定探索性
- 初期这些探索生成的样本差不多都"平庸"，reward 相近
- 训练推进后，策略偏向 reward 高的区域，但探索仍然会产生一些质量低的样本 → 好坏差距变大

**总结**：策略更新拉开差距 + RM 区分放大 + 基线归一化凸显差异 + 探索保证对比度 → reward 方差随训练增大。
